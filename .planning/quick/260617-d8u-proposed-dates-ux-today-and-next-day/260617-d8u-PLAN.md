# Quick Task 260617-d8u: Proposed Dates UX — Today at 18:00 + Next-Day Auto-Advance

## Goal

Both the original quest Create form and the follow-up CreateFollowUp form should make date entry frictionless:
1. The first date defaults to today at 18:00 (currently defaults to tomorrow in the original form; empty in the follow-up form).
2. The follow-up form starts with one date already filled in (currently user must click "Add Date" to get the first one).
3. Each "Add Date" click advances the date by exactly 1 day from the last filled date at 18:00 (currently adds an empty field or always uses tomorrow).

## Task 1: Fix site.js date defaults

**Files:** `EuphoriaInn.Service/wwwroot/js/site.js`

**Actions:**
1. In `setDefaultDateTime(input)`: change the date to today (not tomorrow). Remove the `+1 day` offset — set `now.setHours(18, 0, 0, 0)` and format `now` directly.
2. In `addProposedDate()`: replace the `setDefaultDateTime(newInput)` call with last-date+1-day logic:
   - Query all datetime-local inputs in `#proposed-dates` that were there BEFORE the new one was appended (i.e., skip the last one which is the newly added input).
   - Find the last one with a non-empty value and parse its date.
   - If found: set new input value to that date + 1 day at 18:00.
   - If not found: set new input value to today at 18:00.
   - Format as `YYYY-MM-DDTHH:MM`.

**Verify:** `setDefaultDateTime` no longer references `tomorrow` / `getDate() + 1`. `addProposedDate` no longer calls `setDefaultDateTime`.

**Done:** Both functions updated, build succeeds (no compilation step needed for JS, just no syntax errors).

## Task 2: Fix CreateFollowUp.cshtml date defaults + auto-init

**Files:** `EuphoriaInn.Service/Views/Quest/CreateFollowUp.cshtml`

**Actions:**
1. In `addDate()`: after appending the entry and before incrementing `dateCount`, add next-date logic:
   - Query all existing datetime-local inputs in `#proposed-dates` (the container now includes the new one — pick the second-to-last, i.e., `allInputs[allInputs.length - 2]` if length >= 2).
   - If that input has a value, parse it and set new input to that date + 1 day at 18:00.
   - Otherwise: set new input to today at 18:00.
   - Format as `YYYY-MM-DDTHH:MM`.
2. Add a `DOMContentLoaded` listener (or extend existing `@section Scripts` inline script):
   - If `#proposed-dates` contains zero `.proposed-date-entry` elements, call `addDate()` once.
   - This ensures the follow-up form starts with one date pre-filled (today at 18:00).

**Verify:** The `addDate()` function sets the new input's value. A DOMContentLoaded handler calls addDate() when container is empty.

**Done:** Follow-up form renders with one pre-filled date on page load. Adding additional dates advances by 1 day each time.
