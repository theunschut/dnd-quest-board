**Status:** ✅ SHIPPED 2026-06-28

**Total Plans:** 22
**Git tag:** v4.0.0

## Overview

Milestone 4 adds three capabilities to the existing application: styled HTML email templates for all outbound notifications, automated and DM-triggered session reminders via Hangfire, and an admin email stats dashboard backed by the Resend REST API. The existing email delivery path (SmtpClient → Postfix → Resend SMTP relay) is unchanged — only the template rendering and job orchestration layers are new.

The critical infrastructure decision is `HtmlRenderer` (built into .NET 10) for template rendering inside background jobs — `IRazorViewEngine` throws `NullReferenceException` in that context. Every Hangfire job resolves scoped services via `IServiceScopeFactory` to avoid DbContext lifetime violations.

## Milestone Summary

**Key Accomplishments:**
1. Hangfire background job infrastructure with SQL Server storage and admin-only dashboard
2. All outbound emails upgraded from plain text to styled HTML via Razor components + HtmlRenderer
3. Quest finalization dedup guard prevents duplicate emails on re-finalization
4. 24h automated session reminders via daily Hangfire cron + DM manual trigger
5. Admin email stats dashboard (sent/delivered/bounced/failed) from Resend REST API
6. Email confirmation flow: admin resend button + job guards + confirmation callback

**Deferred (documented):**
- EMAIL-04 / REMIND-02: Digest batching (single combined email for multiple same-day quests) — dropped because same-day quests have never occurred in one year; revisit when scheduling density increases

**Key Decisions:**
- `HtmlRenderer` (not `IRazorViewEngine`) for background job email rendering — IRazorViewEngine throws NullReferenceException outside request context
- `IServiceScopeFactory` + `CreateAsyncScope()` inside every Hangfire job — scoped services (DbContext, IEmailService) cannot be constructor-injected in jobs
- `IDashboardAuthorizationFilter` (not `LocalRequestsOnlyAuthorizationFilter`) for Hangfire dashboard — Docker reverse proxy bypasses localhost check
- `NullObject dispatchers` for Testing environment isolation — Hangfire not registered in Testing; NullQuestEmailDispatcher / NullReminderJobDispatcher satisfy interface contracts without Hangfire dependency
- Resend API via plain `HttpClient` (no SDK) — avoids package dependency for a read-only stats endpoint

**Technical Debt Incurred:**
- `NoOpBackgroundJobClient` stub in test factory registered alongside `NullQuestEmailDispatcher` / `NullReminderJobDispatcher` — AdminController takes `IBackgroundJobClient` directly (fixed in test-fix commit on 2026-06-28)
- Phase 23 SUMMARY notes 4 pre-existing mobile test failures (now fixed in the same test-fix commit)
