# Milestone v3.0: Mobile Version

**Status:** ✅ SHIPPED 2026-06-25
**Phases:** 12–19
**Total Plans:** 34

## Overview

Milestone 3 delivered a purpose-built mobile experience by adding `.Mobile.cshtml` view variants alongside all existing desktop views. No controllers, ViewModels, repositories, or domain services were modified — every change was strictly additive to the Service layer's Views directory and static assets. A single middleware + view-expander infrastructure block (Phase 12) landed first; once in place each subsequent phase delivered independently verifiable mobile views.

## Phases

### Phase 12: Mobile Infrastructure

**Goal**: The mobile detection pipeline is live; every mobile request routes through `_Layout.Mobile.cshtml` and can resolve `.Mobile.cshtml` view variants with no action required in individual views or controllers
**Depends on**: Nothing (first Milestone 3 phase)
**Requirements**: INFRA-01, INFRA-02, INFRA-03, INFRA-04, INFRA-05, INFRA-06
**Plans**: 3/3 complete — completed 2026-06-24

Plans:
- [x] 12-01-PLAN.md — MobileDetectionMiddleware + MobileViewLocationExpander + Program.cs registration
- [x] 12-02-PLAN.md — _Layout.Mobile.cshtml + _ViewStart conditional routing + INFRA-02/04/05 integration tests
- [x] 12-03-PLAN.md — mobile.css baseline (44px touch targets) + MobileCssTests

**Details:** `MobileDetectionMiddleware` sets a request flag based on User-Agent string. `MobileViewLocationExpander` inserts `.Mobile.cshtml` ahead of the default view path when the flag is set. `_Layout.Mobile.cshtml` provides the offcanvas nav shell. `mobile.css` enforces 44px minimum touch targets. All INFRA-01 through INFRA-06 requirements satisfied.

## Milestone Summary

**Key Accomplishments:**
1. Mobile detection middleware + view-location expander — zero changes to controllers or ViewModels
2. 19 new `.Mobile.cshtml` view files + 14 dedicated mobile CSS files
3. 107 integration tests green at Phase 14 completion; 139 at milestone close
4. Agenda-style calendar replacing 7-column grid on mobile
5. Per-date vote buttons in `_Calendar.Mobile.cshtml` partial replacing simplified Phase 13 voting block

**Key Decisions:**
- Strictly additive view layer approach — no controller or ViewModel changes
- `.Mobile.cshtml` naming convention — resolved automatically by MobileViewLocationExpander
- `mobile.css` baseline with 44px touch targets — enforced site-wide before any view-specific CSS
