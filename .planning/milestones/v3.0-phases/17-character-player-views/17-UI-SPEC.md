## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline icon padding (e.g. `me-1` on Font Awesome icons) |
| sm | 8px | Compact element spacing, badge gaps (`me-2 mb-2` on class badges) |
| md | 12px | Glass card internal padding (`padding: 12px` — established in quests.mobile.css) |
| lg | 16px | Form group spacing (`mb-3` = 1rem = 16px), section gaps |
| xl | 24px | Section break padding between stacked glass cards |
| 2xl | 32px | Major vertical breaks between page sections |
| touch | 44px | Minimum height for all interactive elements (buttons, inputs, selects, textareas) |

Exceptions:
- Touch target minimum 44px overrides all interactive element sizing — enforced globally by `mobile.css` via `.btn, input, select, textarea, .form-control, .form-select { min-height: 44px }`.
- Portrait image capped at `max-height: 220px` (D-02); this is a display constraint, not a spacing token.
- Glass card `border-radius: 12px` — display property, not spacing token.

> Source: mobile.css baseline (INFRA-06); quests.mobile.css established `padding: 12px` for glass cards; D-14 confirms per-page CSS extends baseline without redefining it.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label / meta | 14px (0.875rem) | 400 (regular) | 1.3 |
| Section heading | 13.6px (0.85rem) | 700 (bold, uppercase) | 1.2 |
| Display / nav brand | 24px (1.5rem) | 700 (bold, Cinzel) | 1.2 |

Notes:
- `body { font-size: 16px }` is set in `mobile.css` to prevent iOS auto-zoom on input focus.
- Label/meta size (0.875rem) established in `guild-members.mobile.css` (`.guild-member-class`, `.guild-member-owner`) and `quests.mobile.css` (`.calendar-date-label-mobile`).
- Section headings use `font-size: 0.85rem; text-transform: uppercase; letter-spacing: 0.05em` — established by `.quest-section-heading` and `.guild-section-heading`.
- Character name on Details page renders at `<h2>` — Bootstrap default ~2rem; parchment text treatment overrides color.
- All text inside glass cards uses parchment color override (see Color section).

> Source: mobile.css; quests.mobile.css; guild-members.mobile.css.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Notice Board texture (`/images/Notice Board-blank2.jpg`) | Full-viewport background fixed; warm tan/brown |
| Secondary (30%) | `rgba(255, 255, 255, 0.15)` with `backdrop-filter: blur(15px)` | Glass card containers — all content cards in this phase |
| Accent (10%) | `#F4E4BC` (parchment) | All text inside glass cards; nav brand; offcanvas nav links |
| Accent secondary | `rgba(244, 228, 188, 0.7)` | Muted/secondary text inside glass cards (class/owner labels, empty state body) |
| Destructive | Bootstrap `btn-danger` (`#dc3545`) | Delete Character button only |

Glass card border: `1px solid rgba(255, 255, 255, 0.3)`
Glass card shadow: `0 8px 32px rgba(0, 0, 0, 0.2)`
Row dividers inside cards: `1px solid rgba(139, 69, 19, 0.3)` (warm brown)

Accent (`#F4E4BC`) is reserved for:
- All text content inside glass card containers (headings, body, labels)
- Navbar brand
- Offcanvas navigation links
- Section heading labels (uppercase, 0.85rem)
- Character name and owner name on Details page

Text outside glass cards (on bare notice-board background) uses dark color: `h1–h6 { color: #1a0f08 }` with white text-shadow — established in `mobile.css`.

Parchment text-shadow (full): `text-shadow: 2px 2px 4px rgba(0,0,0,0.9), -1px -1px 2px rgba(0,0,0,0.9)`
Parchment text-shadow (muted variant): `text-shadow: 1px 1px 3px rgba(0,0,0,0.9)`

Status badge colors (inside glass cards, no text-shadow on badges):
- Active: `badge bg-success` (Bootstrap green)
- Retired: `badge bg-secondary` (Bootstrap grey)
- Main Character: `badge bg-warning` (Bootstrap yellow)

Button semantic colors (filled, not outline — CLAUDE.md requirement):
- Edit Character: `btn-warning` (amber)
- Retire Character: `btn-secondary` (grey)
- Reactivate Character: `btn-success` (green)
- Delete Character: `btn-danger` (red)
- Save Changes / Create Character: `btn-warning` (amber — matches desktop Create/Edit)
- Cancel / Back: `btn-secondary` (grey)
- Add Another Class: `btn-sm btn-secondary`
- Remove class entry: `btn-danger`

> Source: D-04 (glass card + parchment treatment); quests.mobile.css; guild-members.mobile.css; mobile.css; Details.cshtml desktop button colors.

## View-by-View Interaction Contract

### GuildMembers/Details.Mobile.cshtml

**Layout order (D-01, single column):**
1. Portrait card (glass card): portrait image or `fa-user` icon placeholder → character name (`<h2>`) → owner name → status badges (Active/Retired) + role badge (Main Character)
2. Character Information card (glass card): Level → Sheet Link → Class badges loop → Description → Backstory
3. Owner Actions card (glass card, visible only if `isOwner`): Edit Character (`btn-warning`) → Retire/Reactivate (`btn-secondary`/`btn-success`) → Delete (`btn-danger`)
4. Back button row (outside glass cards): "Back to Guild Members" (`btn-secondary`)

**Portrait (D-02):**
- If `Model.ProfilePicture != null`: `<img src="@Url.Action("GetProfilePicture", new { id = Model.Id })" class="character-portrait-mobile img-fluid" style="max-height: 220px; width: auto;" />` — centered in card (`text-center`)
- Else: `<i class="fas fa-user fa-5x"></i>` with parchment color — centered, no border/box

**Class badges (Claude's Discretion):**
`<span class="badge bg-primary me-1 mb-1">@charClass.Class Level @charClass.ClassLevel</span>` — one badge per class, naturally wrapping on narrow screens. Text-shadow: none on badges.

**Sheet link (Claude's Discretion):**
Renders with `class="text-break"` — allows natural line wrap at any character. No truncation via `text-overflow: ellipsis`.

**Owner actions (D-03):**
- All three buttons rendered `w-100` (full-width) with 8px bottom margin (`mb-2`) between them
- Delete uses `onsubmit="return confirm('...')"` — native browser dialog, no custom modal
- ToggleRetirement uses POST form — no JS confirmation

**CSS class names for this view:**
- Glass card container: `character-detail-card` (in `character-detail.mobile.css`)
- Portrait image: `character-portrait-mobile`
- Section heading labels: `character-section-heading` (uppercase, 0.85rem, parchment)
- Muted text: `character-muted-text` (rgba(244,228,188,0.7))

## CSS Architecture Contract

Three new CSS files (D-12):

| File | Loaded By | Scope |
|------|-----------|-------|
| `character-detail.mobile.css` | `GuildMembers/Details.Mobile.cshtml` | Portrait sizing, section spacing, action button layout |
| `character-form.mobile.css` | `GuildMembers/Create.Mobile.cshtml` AND `GuildMembers/Edit.Mobile.cshtml` | Class-entry stacked layout, file input block, form container glass card |
| `players.mobile.css` | `Players/Index.Mobile.cshtml` | Section glass cards, name list rows, empty state |

Each file follows the established pattern (D-14):
- No `@media` queries — file is only loaded by the mobile view
- No redefinition of `mobile.css` baseline rules (44px touch targets, body font-size, h1–h6 color)
- Glass card rule set copied from `quests.mobile.css` / `guild-members.mobile.css`, renamed per-page
- Parchment text copied from established selectors

**CSS injection pattern per view (D-13):**
```html
@section Styles {
    <link href="~/css/{filename}.css" asp-append-version="true" rel="stylesheet" />
}
```

## Established Patterns — Quick Reference for Executor

All patterns below are authoritative; verified against Phases 13–16 implementations.

**Glass card CSS (copy verbatim, rename selector):**
```css
.{page}-card {
    background: rgba(255, 255, 255, 0.15);
    backdrop-filter: blur(15px);
    border: 1px solid rgba(255, 255, 255, 0.3);
    border-radius: 12px;
    box-shadow: 0 8px 32px rgba(0, 0, 0, 0.2);
    padding: 12px;
}
```

**Parchment text (copy verbatim):**
```css
color: #F4E4BC !important;
text-shadow: 2px 2px 4px rgba(0,0,0,0.9), -1px -1px 2px rgba(0,0,0,0.9);
```

**Parchment muted text:**
```css
color: rgba(244, 228, 188, 0.7) !important;
text-shadow: 1px 1px 3px rgba(0,0,0,0.9) !important;
```

**Section heading:**
```css
text-transform: uppercase;
font-size: 0.875rem;
letter-spacing: 0.05em;
```

**Tap navigation row:**
```html
<div class="d-flex align-items-center p-3"
     onclick="window.location.href='@Url.Action(...)'">
```

**No `@inject` in mobile views** — `IAntiforgery`, `IAuthorizationService` etc. available globally via `_ViewImports.cshtml`. Do not add `@inject` declarations.

**No `Layout` property in mobile views** — `_ViewStart.cshtml` handles layout selection. Do not set `Layout = ...` in mobile views.

**No `@{}` wrapper for variable assignment inside foreach** — assign variables directly in C# code mode (Phase 13 pattern enforced).

**Razor `@section Scripts` renders unconditionally** — place outside of conditional blocks to ensure JS is available even when controlling HTML is hidden.
