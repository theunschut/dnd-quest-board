## Design System

| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`me-1`, `me-2`), badge padding |
| sm | 8px | Compact row padding, inline element gaps |
| md | 16px | Card padding (`padding: 16px`), form field `mb-3` |
| lg | 24px | Section breaks between glass cards (`mb-4`) |
| xl | 32px | Page-level top/bottom margin on main container |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets: minimum 44px height on all `.btn`, `input`, `select`, `textarea`, `.form-control`, `.form-select` — inherited from `mobile.css` baseline (INFRA-06).
- Guild Members circular thumbnail: 40×40px (`border-radius: 50%`) — deliberate sub-grid size for profile images in list rows.
- DM profile photo pattern if reused for character thumbnails: 40×40px.

Source: CONTEXT.md D-09, existing `mobile.css`, `dm-create.mobile.css`, `dm-profile.mobile.css`.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label / meta | 14px (0.875rem) | 400 (regular) | 1.3 |
| Section heading | 14px (0.875rem), uppercase, letter-spacing 0.05em | 700 (bold) | 1.2 |
| Card / page heading | 20px (h2/h3 default, `mb-0` via Bootstrap) | 700 (bold) | 1.2 |

Notes:
- 16px body is mandatory on mobile — prevents iOS auto-zoom on input focus (set globally in `mobile.css`).
- Parchment text (`#F4E4BC`) applies inside glass cards only. Outside glass cards (notice board background), headings use the dark warm `#1a0f08` with white text-shadow (from `mobile.css` `h1–h6` rule).
- Section headings inside glass cards (e.g. "My Characters", "Password Management") use `font-size: 0.875rem; text-transform: uppercase; letter-spacing: 0.05em` — matching `.quest-section-heading` and `.dm-profile-section-heading` patterns.

Source: `mobile.css` lines 1–10 and 28–33, `dm-profile.mobile.css` lines 49–56, `quests.mobile.css` lines 94–101.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Notice board texture image (`/images/Notice Board-blank2.jpg`) | Full-page background, fixed attachment |
| Secondary (30%) | `rgba(255, 255, 255, 0.15)` with `backdrop-filter: blur(15px)` | Glass card surfaces for all content panels |
| Accent (10%) | `#F4E4BC` (parchment) | Text inside glass cards: headings, body copy, labels |
| Accent — gold | `#FFD700` | Hover state on offcanvas nav links only |
| Accent — primary buttons | `linear-gradient(45deg, #8B4513, #A0522D)` | `.btn-primary` — primary actions (Save, Submit) |
| Accent — warning buttons | Bootstrap `btn-warning` (amber) | Secondary actions: Log in, Create Account, Back to Quest Board, Create New Character, Filter & Sort trigger |
| Destructive | `rgba(220, 53, 69, 0.9)` | `.alert-danger` error states; no destructive action buttons in this phase |

Accent reserved for:
- `#F4E4BC` parchment: ALL text rendered inside glass card containers (headings, form labels, body copy, section labels, character names, item names).
- Gold `#FFD700`: offcanvas nav hover only — do not use on page content.
- Brown gradient: `.btn-primary` submit / save buttons.
- Amber `btn-warning`: navigation and creation CTA buttons (Log in, Create Account, Create New Character).

Glass card border: `1px solid rgba(255, 255, 255, 0.3)`.
Glass card shadow: `box-shadow: 0 8px 32px rgba(0, 0, 0, 0.2)`.
Glass card border-radius: `12px`.

Source: CONTEXT.md D-02, D-07; `quests.mobile.css` lines 5–12; `dm-profile.mobile.css` lines 3–12; `mobile.css` lines 99–120.

## Per-View Layout Contract

### Account Pages (Login, Register, Edit, Profile, ChangePassword)

CSS file: `account.mobile.css` (shared across all five Account mobile views).

**Login.Mobile.cshtml** (satisfies ACCT-01):
- Single glass card, full-width (`col-12` or no column wrapper — remove desktop `col-md-6 col-lg-4`).
- Card header: `fas fa-sign-in-alt` icon + "Log in" title — parchment treatment.
- Form fields: Email, Password (both `form-control`, 44px min-height from baseline), RememberMe checkbox.
- Submit: full-width `d-grid` button with `btn-warning btn-lg` — "Log in".
- Below form: `<hr>` separator, then "Don't have an account?" + "Create Account" `btn-warning` link.
- Validation summary (`asp-validation-summary="All"`) rendered with `.text-danger` inside card body.

**Register.Mobile.cshtml** (satisfies ACCT-02):
- Single glass card, full-width (remove desktop `col-md-6 col-lg-5`).
- Card header: `fas fa-user-plus` icon + "Register" title — parchment treatment.
- Subtitle: "Join the Quest Board community" — faded parchment (`rgba(244, 228, 188, 0.7)`).
- Form fields: Name, Email, Password, ConfirmPassword — all `form-control`, 44px min-height.
- Password hint: `form-text` — faded parchment.
- Submit: full-width `d-grid` button `btn-warning btn-lg` — "Create Account".
- Below form: `<hr>` separator, "Already have an account?" + "Log in" `btn-warning` link.

**Edit.Mobile.cshtml** (satisfies ACCT-03, part 1):
- Single glass card, full-width (remove desktop `col-md-6`).
- Card header: `fas fa-user-edit text-warning` icon + "Edit Profile" title — parchment treatment.
- Form fields: Name, Email (both `form-control`), HasKey checkbox (`form-check-input`).
- Hidden `asp-for="Id"` field preserved verbatim.
- `<hr>` separator before Password Management section.
- Password Management section heading: `0.875rem uppercase` parchment — "Password Management".
- "Change Password" link: `btn-warning` with `fas fa-key` icon (replace desktop `btn-outline-primary` with filled `btn-warning` per CLAUDE.md button guideline).
- `<hr>` separator before action buttons.
- Button row: `d-flex justify-content-between` — Cancel (`btn-secondary`, left) + Save Changes (`btn-success`, right).

**Profile.Mobile.cshtml** (satisfies ACCT-03, part 2):
- Single glass card, full-width (remove desktop `col-md-6`).
- Card header: `fas fa-user-cog text-success` icon + "Profile" title — parchment treatment.
- Profile fields rendered as stacked label-value pairs (not desktop `row col-sm-3 col-sm-9` grid layout — use single column: label above value).
- Field labels: 0.875rem faded parchment (`rgba(244,228,188,0.7)`).
- Field values: 1rem parchment (`#F4E4BC`).
- Account type indicators: preserve `fas fa-shield-alt text-danger` (Admin), `fas fa-crown text-warning` (DM), plain text (Player).
- Building access: preserve `fas fa-key text-success` / `fas fa-times text-muted` icons.
- `<hr>` separator before buttons.
- Success message alert: `alert-success` with `fas fa-check-circle` — rendered in parchment context (do NOT apply `.text-danger` class here).
- Button row: `d-flex justify-content-between` — "Back to Quest Board" (`btn-warning`, left) + "Edit Profile" (`btn-primary`, right).

**ChangePassword.Mobile.cshtml** (satisfies ACCT-03, part 3):
- Single glass card, full-width (remove desktop `col-md-6`).
- Card header: `fas fa-key text-primary` icon + "Change Password" title — parchment treatment.
- Form fields: CurrentPassword, NewPassword, ConfirmPassword — all `form-control`, 44px min-height.
- Validation summary: `asp-validation-summary="All"` with `.text-danger`.
- `<hr>` separator before action buttons.
- Button row: `d-flex justify-content-between` — Cancel (`btn-secondary`, left) + Change Password (`btn-primary`, right).

### Shop Index (Shop/Index.Mobile.cshtml) — satisfies BROWSE-01

CSS file: `shop.mobile.css`.

**Header section:**
- Glass card header: "The Mystical Merchant's Emporium" title + subtitle copy — parchment treatment.
- `fas fa-store` icon in heading.

**Filter & Sort drawer (D-03):**
- A single `btn-warning` "Filter & Sort" trigger button renders at top of page, full-width.
- Icon: `fas fa-filter me-2`.
- Drawer implementation: Bootstrap offcanvas (`offcanvas-bottom`, `data-bs-toggle="offcanvas"`) — simplest available mechanism with no new JS required. Height: `auto` (content-driven, max 70vh).
- Inside offcanvas: category type tabs (All, Equipment, Magic Items, Quest Items) as stacked link buttons; rarity checkboxes; sort dropdown; Search input; Apply Filters submit button; Clear Filters link (conditional).
- Offcanvas background: `rgba(20, 8, 0, 0.97)` matching existing `.offcanvas` rule in `mobile.css`.
- Filter/sort form uses the same `BuildTabUrl` / `BuildPageUrl` `@functions` from desktop view — copy verbatim.
- Active filter indicator: if any filter is active (`Model.SelectedRarities.Any() || Model.SelectedSort != null || !string.IsNullOrEmpty(Model.SearchQuery)`), append a small amber badge count to the "Filter & Sort" button.

**Item grid (D-05):**
- 2-column CSS grid: `display: grid; grid-template-columns: 1fr 1fr; gap: 8px;`.
- Each item card: glass card surface, `border-radius: 12px`, `padding: 8px`.
- Item card contents (top to bottom): item name (parchment, `font-size: 1rem`, truncate at 2 lines), rarity badge (no text-shadow), price (`fas fa-coins` icon + gold value in parchment text), Buy button (`btn-warning btn-sm`, full card width, 44px min-height).
- Item card tap: `data-bs-toggle="modal" data-bs-target="#itemDetailsModal"` + same `data-item-url` attribute as desktop — the existing JS event listener populates the modal without modification. Include the modal markup and JS event listener block verbatim from desktop view (lines ~491–520).
- Purchase History side panel (D-04): omitted entirely on mobile.

**Empty state (no items):**
- Glass card: `fas fa-box-open fa-3x` icon (faded parchment), heading "No items found", body "Try adjusting your filters or search query."

**Pagination:**
- Previous / page numbers / next — rendered as `d-flex justify-content-center gap-2` button row. Each page button: `btn-warning btn-sm` (active page) or `btn-outline-secondary btn-sm` (inactive). Minimum 44px height.

### Guild Members (GuildMembers/Index.Mobile.cshtml) — satisfies BROWSE-02

CSS file: `guild-members.mobile.css`.

**Page layout:**
- No desktop two-column `col-md-6` grid. Both sections (My Characters, Guild Roster) stack vertically, full-width, separated by `mb-4`.
- "Create New Character" button: appears at top of page, above the My Characters section. Full-width `btn-warning` with `fas fa-plus me-2` icon and "Create New Character" label.

**Section structure (D-07):**
- Each section wrapped in a glass card: `background: rgba(255, 255, 255, 0.15)`, `backdrop-filter: blur(15px)`, `border: 1px solid rgba(255, 255, 255, 0.3)`, `border-radius: 12px`, `padding: 12px`.
- Section header inside card: parchment text. "My Characters" (`fas fa-user-circle text-success`), "Guild Roster" (`fas fa-users text-primary`).

**Character list rows (D-06, D-07):**
- Single-column list. One character per row.
- Row structure: `<div class="d-flex align-items-center p-3" onclick="window.location.href='@Url.Action("Details", new { id = character.Id })'">`.
- Left: circular thumbnail 40×40px. If `character.ProfilePicture != null`: `<img src="@Url.Action("GetProfilePicture", new { id = character.Id })" class="guild-member-thumbnail" />` with `border-radius: 50%; object-fit: cover; width: 40px; height: 40px; flex-shrink: 0`. If no picture: placeholder `<div class="guild-member-placeholder"><i class="fas fa-user"></i></div>` (40×40px circle, `rgba(255,255,255,0.1)` background).
- Right (ms-3 flex-grow-1): character name (`font-size: 1rem; color: #F4E4BC`) on first line; class/level string on second line (0.875rem, faded parchment); role and status badges on third line (no text-shadow).
- Other Characters rows: add owner name below class/level (`fas fa-user me-1` icon, faded parchment, `font-size: 0.875rem`).
- Retired character rows: `opacity: 0.7` on the row container to visually dim retired characters.
- Row divider: `border-bottom: 1px solid rgba(139, 69, 19, 0.3)` between rows; last row has no border.
- Tap feedback: `:active { background-color: rgba(255, 255, 255, 0.05); border-radius: 4px; }`.

**Empty states:**
- My Characters empty: glass card body contains `fas fa-users-slash fa-2x` icon (faded parchment, centered), heading "No Characters Yet" (parchment), body "Create your first character to join the guild!" (faded parchment), "Create Character" `btn-warning` button.
- Guild Roster empty: `fas fa-dungeon fa-2x` icon (faded parchment, centered), heading "No Other Characters" (parchment), body "The guild awaits other adventurers!" (faded parchment). No button.

## CSS Architecture

| File | Views Served | Contents |
|------|-------------|---------|
| `account.mobile.css` | Login, Register, Edit, Profile, ChangePassword | Glass card containers, parchment form labels, stacked field layout, full-width `col-12` overrides |

## Integration Contracts

### View Injection Pattern
Every `.Mobile.cshtml` view in this phase includes:
```razor
@section Styles {
    <link href="~/css/{file}.mobile.css" asp-append-version="true" rel="stylesheet" />
}
```
No individual view sets `Layout` — `_ViewStart.cshtml` handles layout selection automatically (Phase 12, INFRA-05).

### Modal Reuse (Shop)
The `#itemDetailsModal` Bootstrap modal and its JS event listener block must be included verbatim from `Shop/Index.cshtml`. The event listener reads `data-item-url` from item card `data-*` attributes and fetches the modal body via AJAX. Mobile item cards must carry identical `data-item-url` attributes.

### No @inject in Mobile Views
`IAntiforgery`, `IAuthorizationService`, and `IUserService` are globally injected via `_ViewImports.cshtml`. Do not add `@inject` directives in any `.Mobile.cshtml` view — doing so causes a duplicate injection compile error.

### No @{} Wrapper in Foreach Bodies
Declare C# variables directly in code mode inside `@foreach{}` bodies without an `@{}` wrapper — Phase 13/14/15 enforced pattern.

### Filter Form State Preservation (Shop)
Hidden input fields for `SelectedType`, `SelectedRarities`, `SelectedSort`, and `SearchQuery` must be preserved in the mobile filter form so pagination (`BuildPageUrl`) and tab navigation (`BuildTabUrl`) carry current filter state. Copy the `@functions` block from the desktop view verbatim.
