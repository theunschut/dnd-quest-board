<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → AdminController.EmailStats | Untrusted HTTP request crosses into the admin-only action; the `force` query param is the only user-controlled input |
| AdminController → Resend API | The `ResendApiKey` bearer secret and outbound request cross to an external SaaS |
| Resend API → AdminController | Untrusted JSON response is deserialized into `ResendEmailListResponse` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-04 | Elevation of Privilege | `/Admin/EmailStats` route | mitigate | Existing class-level `[Authorize(Policy = "AdminOnly")]` covers the new action (D-11); two integration tests in Task 3 assert unauthenticated and non-admin access is denied (ASVS V4). |
| T-23-05 | Information Disclosure | API failure / exception path | mitigate | `GetResendStatsAsync` wraps the call in try/catch and returns a generic `ApiError()` view (D-06); the `apiKey` and exception text are never written to the response or logs. |
| T-23-06 | Denial of Service (Resend rate limit) | repeated stat fetches | mitigate | 5-minute `IMemoryCache` TTL (D-07); a fresh fetch only happens on cache miss or explicit `?force=true` admin action (D-08). |
| T-23-07 | Input Validation | `force` query param | accept | `force` binds as `bool` (model binding rejects non-bool); no injection surface — value only toggles a cache check. ASVS V5 low risk. |
| T-23-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase (RESEARCH.md Package Legitimacy Audit: none). No blocking-human checkpoint required. |
</threat_model>
