## Copywriting Contract

| Element | Copy |
|---------|------|
| Nav link label | "Background Jobs" |
| Nav link icon | `fas fa-tasks` (FontAwesome — tasks/checklist icon) |
| Primary CTA | Not applicable — this phase has no form submissions or CTAs |
| Empty state heading | Not applicable — Hangfire dashboard provides its own built-in empty state |
| Empty state body | Not applicable — third-party dashboard UI |
| Error state | Not applicable — authorization failure redirects to existing `/Account/Login` (D-02 from CONTEXT.md); no custom error UI added |
| Destructive confirmation | Not applicable — no destructive actions in this phase |

**Nav link rationale (D-10, Claude's Discretion):** "Background Jobs" is preferred over "Hangfire" (too implementation-specific for an admin nav label) and over "Job Dashboard" (redundant noun). The `fa-tasks` icon visually communicates job queues to an Admin without requiring domain knowledge of Hangfire. Placement is as the third item in the Admin dropdown, after "Quest Management" — consistent with admin panel flow from user-facing management to infrastructure.

## Component Inventory

The only new markup element in this phase:

```html
@* In _Layout.cshtml — Admin dropdown <ul> (after Quest Management <li>) *@
<li>
    <a class="dropdown-item" href="/hangfire">
        <i class="fas fa-tasks me-2"></i>Background Jobs
    </a>
</li>
```

Rules:
- Use plain `href="/hangfire"` — NOT `asp-controller`/`asp-action` tag helpers (Hangfire is a middleware route, not an MVC action).
- The `me-2` class (8px right margin) is the established icon-to-label spacing in all existing dropdown items — maintain consistency.
- No `<li><hr class="dropdown-divider"></li>` separator before this item — it is a peer item to User Management and Quest Management, not a different section.
- Visibility: inherits the existing `@if ((await AuthorizationService.AuthorizeAsync(User, "AdminOnly")).Succeeded)` condition wrapping the entire Admin dropdown — no additional role check on this `<li>`.
