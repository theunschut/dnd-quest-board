<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DB content → Razor template → HTML string | Quest title, description, DM name, player names flow from database into @variable rendering — auto-escaped by Razor |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-04 | Tampering | QuestDescription / QuestTitle rendered into email HTML | mitigate | Razor `@variable` syntax auto-escapes HTML entities. `@QuestDescription` renders as escaped text. Never wrap user content in `@((MarkupString)...)` — this would be XSS. |
| T-21-05 | Tampering | ConfirmedPlayerNames in `string.Join(", ", ...)` call | mitigate | Each name rendered via `@string.Join(...)` is auto-escaped. No MarkupString. |
| T-21-06 | Spoofing | AppUrl in image src / background-image | accept | AppUrl is server-configured from EmailSettings (appsettings.json); not user-supplied. |
</threat_model>
