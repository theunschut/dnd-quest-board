# Phase 21 — UI Design Contract: HTML Email Templates

> Visual contract for inline-CSS Razor email components. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> This phase produces email HTML, not web pages. No component library, no Tailwind, no shadcn.
> All styles are INLINE — email clients strip external stylesheets.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — inline CSS in Razor components |
| Preset | not applicable |
| Component library | none — HtmlRenderer + Razor components (.razor files) |
| Icon library | none — decorative elements are images (wax seal, CR badge) |
| Heading font | Cinzel — loaded via `<link>` from Google Fonts in `<head>` |
| Body font | Georgia, serif fallback |
| Rendering engine | `HtmlRenderer` (Microsoft.AspNetCore.Components.Web, .NET 10) |
| CSS delivery | Inline `style=""` attributes only — never `<style>` blocks or external files |

**Google Fonts link (placed in `<head>` of `_EmailLayout.razor`):**
```
https://fonts.googleapis.com/css2?family=Cinzel:wght@400;700&display=swap
```
Outlook ignores this link and falls back to `serif` — acceptable for the 17-member group (all on Gmail / Apple Mail / iOS Mail, per D-02).

## Layout Structure

All emails use a table-based layout (required for email client compatibility):

```
<body>                          background-color: #F4E4BC (fallback for Outlook)
  <table width="600">           centered, max-width 600px
    <tr>
      <td>                      background-image: Poster1.png or Poster6.png
                                background-size: cover
                                background-position: center top
        <!-- CR badge: top-left absolute-style positioning via nested table -->
        <!-- Quest title: centered Cinzel heading -->
        <!-- Description: italic body text -->
        <!-- DM / Date / Players: bold label rows -->
        <!-- CTA button: centered gold button -->
        <!-- Wax seal: bottom-left via nested table -->
      </td>
    </tr>
  </table>
</body>
```

**Poster image URLs** are absolute (email clients cannot resolve relative paths):
- Finalized / Reminder: `{AppUrl}/images/Blanks/Blanks w Shadow/Poster1.png`
- Date Changed: `{AppUrl}/images/Blanks/Blanks w Shadow/Poster6.png`
- `AppUrl` is read from `EmailSettings.AppUrl` (already in `EuphoriaInn.Domain/Models/EmailSettings.cs`)

**Wax seal image URL:** `{AppUrl}/images/Wax Seals/{SealFilename}`
- Default seal: `Crown Seal.png` — fixed selection for all email types
- Discretion note (from CONTEXT.md): randomized-per-quest-ID hash is allowed but not required; Crown Seal is the safe default

## Spacing Scale

Email spacing uses px values in inline styles. Multiples of 4, email-appropriate magnitudes.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gap inside CR badge |
| sm | 8px | Padding between label rows (DM / Date / Players) |
| md | 16px | Horizontal content padding inside poster |
| lg | 24px | Vertical padding above/below quest title |
| xl | 32px | Top padding from poster edge to content |
| 2xl | 48px | Space reserved for CR badge clearance at top |
| 3xl | 64px | Bottom padding before wax seal |

Exceptions: The poster image intrinsically defines outer dimensions. Content is overlaid via padding only — no CSS `position: absolute` (most email clients ignore it). Use nested `<table>` cells with `valign` and `padding` to achieve positional effects.

## Typography

All font sizes in px. All styles inline.

| Role | Element | Size | Weight | Font Family | Line Height | Color |
|------|---------|------|--------|-------------|-------------|-------|
| Quest title | `<h1>` or `<td>` | 28px | 700 (bold) | 'Cinzel', serif | 1.2 | #1a0f08 |
| Section heading | `<h2>` or `<td>` | 20px | 700 (bold) | 'Cinzel', serif | 1.2 | #1a0f08 |
| Body / description | `<p>` or `<td>` | 15px | 400 (normal) | Georgia, serif | 1.6 | #1a0f08 |
| Label (DM / Date / Players) | `<td>` | 14px | 700 (bold) | Georgia, serif | 1.4 | #1a0f08 |

**Description text** (QuestFinalized + SessionReminder): rendered italic (`font-style: italic`) at 15px / weight 400.

**Character name list** (SessionReminder only): rendered as comma-separated inline text or a simple unordered list at 14px / weight 400. No table required.

Text shadow for readability against parchment background:
```
text-shadow: 2px 2px 4px rgba(255,255,255,0.9), 1px 1px 6px rgba(0,0,0,0.5)
```
Apply to all text rendered over the poster image.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F4E4BC` | Body background, Outlook fallback `background-color`, content area fill |
| Secondary (30%) | `#2c1810` (dark brown) | Content area subtle overlays, DM/date/player label rows |
| Primary text | `#1a0f08` | All body copy, labels, headings |
| Accent gold (10%) | `#FFD700` | CR badge border, CR badge text, CTA button background, decorative divider lines |
| Accent warm gold | `#ffc107` | CTA button hover fallback (static emails; use #FFD700) |
| Destructive | n/a | No destructive actions in email templates |

**Accent reserved for:**
1. CR badge border (`border: 2px solid #FFD700`)
2. CR badge text color
3. CTA button background (`background-color: #FFD700`)
4. Decorative horizontal rule / divider between description and metadata rows

**CR Badge inline style** (matches `.cr-badge` in quests.css exactly):
```
background: linear-gradient(135deg, #8B4513, #A0522D);
color: #FFD700;
font-weight: bold;
font-size: 14px;
padding: 8px 12px;
border-radius: 8px;
border: 2px solid #FFD700;
box-shadow: 0 4px 8px rgba(0,0,0,0.3);
text-shadow: 1px 1px 2px rgba(0,0,0,0.7);
display: inline-block;
```

## CTA Button Style

```
background-color: #FFD700;
color: #1a0f08;
font-family: 'Cinzel', serif;
font-size: 15px;
font-weight: bold;
padding: 12px 24px;
border-radius: 6px;
text-decoration: none;
display: inline-block;
border: 2px solid #8B4513;
text-align: center;
```

> `font-size: 15px` maps to the **body** tier in the Typography table above. Using body scale keeps the button readable and consistent with body text rhythm — 4 distinct sizes total (28px, 20px, 15px, 14px).

Render as `<a href="{QuestUrl}" style="...">View Quest Details</a>` — never `<button>`. Email clients do not support form elements reliably.
