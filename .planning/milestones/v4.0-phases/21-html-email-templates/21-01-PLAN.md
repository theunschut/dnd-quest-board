## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Quest data → email content | Quest title, description, DM name from DB are rendered into email HTML via Razor `@variable` syntax — auto-escaped |
| EmailSettings.AppUrl → image URLs | AppUrl is server-configured in appsettings.json, not user-supplied — no SSRF risk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-01 | Tampering | EmailService.SendAsync — htmlBody parameter | mitigate | htmlBody is produced by RazorEmailRenderService (Wave 3), not user input; Razor `@variable` auto-escapes. Never use `MarkupString` for user content. |
| T-21-02 | Tampering | MailMessage.Subject — subject parameter | accept | System.Net.Mail sanitizes headers automatically; SMTP protocol cannot inject via Subject in .NET's implementation |
| T-21-03 | Spoofing | AppUrl in image URLs | accept | AppUrl is read from appsettings.json (server config); not user-supplied. No user can influence the constructed URL. |
</threat_model>
