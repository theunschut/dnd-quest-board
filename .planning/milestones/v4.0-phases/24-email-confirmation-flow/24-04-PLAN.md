Call `var result = await identityService.ConfirmEmailAsync(userId, decodedToken);`. If `result.Succeeded`, set `TempData["Success"] = "Email confirmed — you can now log in.";` else set `TempData["Error"] = "Email confirmation failed. The link may be expired or invalid. Contact an administrator.";`. Always `return RedirectToAction(nameof(Login));` (never redirect to a request-supplied URL — fixed redirect target prevents open-redirect). Do NOT add `[Authorize]` — the callback must be reachable by an unauthenticated user clicking the link in their email.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| email client → GET /Account/ConfirmEmail | Untrusted query-string input (`userId`, `token`) from a link clicked in an external email arrives at an unauthenticated endpoint |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-02 | Tampering | Unvalidated userId/token in callback | mitigate | `identityService.ConfirmEmailAsync` (Identity) validates the token is bound to that userId and security stamp; a mismatched userId/token returns failure, not confirmation. No manual cross-check needed. |
| T-24-07 | Repudiation | Token replay on callback | mitigate | Identity tokens are single-use; a second click after success returns `result.Succeeded == false` → error banner. |
| T-24-09 | Elevation of Privilege | Open redirect after confirm | mitigate | Always `RedirectToAction(nameof(Login))` — never redirect to a URL taken from the request, so a crafted link cannot bounce the user to an attacker domain. |
| T-24-10 | Denial of Service / Robustness | Malformed Base64Url token | mitigate | Empty-token guard short-circuits before decode; a malformed token yields a failed `IdentityResult` → error banner rather than a 500. (Executor should confirm `Base64UrlDecode` of a malformed value surfaces as an error banner, not an unhandled exception — wrap the decode if needed.) |
| T-24-SC | Tampering | npm/pip/cargo installs | accept | No package installs — `WebEncoders` and `IIdentityService` already available. No audit table required. |
</threat_model>
