# Milestone v1.0: Architecture & Features

**Status:** ✅ SHIPPED (prior to 2026-06)
**Phases:** 1–7, 9 (Phase 8 deferred)
**Total Plans:** 19

## Overview

Milestone 1 fixed the codebase's accumulated architectural drift before delivering four backlog features. The first four phases restored correct layer boundaries, consolidated business logic into services, removed dead code, and closed security gaps. The remaining phases delivered shop filtering, follow-up quest creation, a DM profile page, and server-side shop pagination on the cleaned-up architecture.

Phase 8 (profile picture avatar crop) was scoped in this milestone but deferred due to unverified SkiaSharp native library availability on the deployment host.

## Phases

### Phase 1: Layer Dependency Fix

**Goal**: The Domain project compiles without any reference to the Repository project; the correct dependency direction (Service → Domain ← Repository) is enforced at build time
**Depends on**: Nothing (first phase)
**Requirements**: ARCH-01, ARCH-02, ARCH-03, ARCH-04
**Plans**: 2/2 complete

Plans:
- [x] 01-01-PLAN.md — Repository infrastructure: move EntityProfile, refactor BaseRepository to dual-generic with IMapper
- [x] 01-02-PLAN.md — Complete dependency inversion: move interfaces to Domain, refactor services, remove ProjectReference

**Details:** `EntityProfile.cs` (AutoMapper Entity↔DomainModel mappings) moved from Domain to Repository. Repository interfaces moved to Domain. `Program.cs` registers AutoMapper profiles by explicit type reference — no assembly scanning. Domain.csproj `<ProjectReference>` to Repository removed.

- [x] 02-02-PLAN.md — QuestService email consolidation: inject IEmailService, slim QuestController, post-save re-fetch

### Phase 4: Security Hardening

**Goal**: Failed login attempts are rate-limited, minimum password length meets the 8-character standard, HasKey is admin-only, the Password property is removed from the domain model, and .env is not tracked by git
**Depends on**: Phase 3
**Requirements**: SEC-01, SEC-02, SEC-03, SEC-04, SEC-05, SEC-06
**Plans**: 4/4 complete

Plans:
- [x] 04-01-PLAN.md — Account lockout: enable lockoutOnFailure, 5-attempt threshold, 15-minute window, EF Core migration for existing users
- [x] 04-02-PLAN.md — Password policy: raise minimum length to 8 characters
- [x] 04-03-PLAN.md — HasKey admin-only: remove from Account/Edit view; keep in Admin panel only; remove Password from domain model
- [x] 04-04-PLAN.md — .env gitignore: add .env to .gitignore, keep .env.example tracked

**Details:** `lockoutOnFailure: true`, 5 attempts, 15-min lock. All existing `AspNetUsers` rows backfilled with `LockoutEnabled = 1` via EF Core migration. Minimum password length raised to 8. `HasKey` removed from user-facing Account/Edit view (remains informational in Admin panel). `Password` property removed from `User` domain model. `.env` added to `.gitignore`.
