## Spacing Scale

Standard Bootstrap 5 utility classes map to the 8-point base scale. All new markup must use Bootstrap spacing utilities exclusively; no inline `style="margin:..."` values.

| Token | Value | Bootstrap utility | Usage |
|-------|-------|-------------------|-------|
| xs | 4px | `me-1`, `mb-1` | Icon-to-text gap within a label |
| sm | 8px | `me-2`, `mb-2` | Icon-to-text gap in buttons, badge spacing |
| md | 16px | `mb-3` | Form field bottom margin, paragraph spacing |
| lg | 24px | `mb-4` | Section separation within a card body |
| xl | 32px | `mb-5` | Major section breaks |
| 2xl | 48px | `py-4` | Card vertical padding (existing pattern) |
| 3xl | 64px | not used in this phase | — |

Exceptions: none for this phase.

Inherits the global typography contract already enforced in `site.css` and `quests.css`. No new type tokens are introduced by this phase; all new text uses the sizes already declared for the quest management views.

| Role | Size | Weight | Line Height | Font | Used where |

| Body | 16px (1rem) | 500 | 1.5 | system sans-serif | Quest Summary sidebar paragraphs (`<p>`) |

## Color

The project uses no design tokens or CSS custom properties for color (aside from `--card-width`). All color values are Bootstrap semantic class names or the project's custom hex values established in `site.css`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | transparent / `rgba(255,255,255,0.15)` backdrop | Page background (notice board image), `.modern-card` glass surface |
| Secondary (30%) | `rgba(255,255,255,0.05)` | `.modern-card-header`, `.modern-card-body` within sidebar cards |
| Accent (10%) | `#8B4513` / `#A0522D` (saddle brown gradient) | Follow-Up button (`btn-primary`) fill only — see reserved list |
| Destructive | `#e74c3c` / `#c0392b` (red gradient) | `btn-danger` — delete and remove actions only |

**Accent reserved for:**
- The "Create Follow-Up Quest" `btn-primary` button on Manage.cshtml
- The `fa-scroll` icon color in the "Continues in/from" sidebar line (use `text-warning` = `#ffc107` to match the existing quest chain icon style consistent with `cr-badge` and quest log meta icons)

**Accent NOT used for:**
- The "Continues in/from" link text itself (use default anchor color, inherited from Bootstrap)
- The follow-up creation form submit button uses `btn-success` (green gradient) following the existing "Finalize Quest" button pattern on Manage.cshtml
- Sidebar link text for related quest title (clickable `<a>` tag, unstyled or default Bootstrap link color)

## Component Inventory

### 1. "Create Follow-Up Quest" Button — Manage.cshtml

**Location:** Inside the finalized state block (`@if (Model.IsFinalized)`), after the existing `<hr>` and selected participants table, above or alongside the existing "Open Quest" / "Refresh Data" button row.

**Exact markup pattern:**
```html
@if (Model.IsFinalized && Model.FollowUpQuest == null)
{
    <a href="@Url.Action("CreateFollowUp", "Quest", new { id = Model.Id })"
       class="btn btn-primary">
        <i class="fas fa-scroll me-2"></i>
        Create Follow-Up Quest
    </a>
}
```

**Visibility rules (from D-10, D-11):**
- Shown only on Manage page, never on Details page
- Hidden (the entire `<a>` tag is not rendered) once `Model.FollowUpQuest != null`
- Requires `[Authorize(Policy = "DungeonMasterOnly")]` on the controller action

**Placement within the finalized button row:**
Add the button to the left side of the `d-flex justify-content-between` row where "Open Quest" currently lives, alongside or replacing the empty left slot. Final row reads:
```
[ Open Quest ]  [ Create Follow-Up Quest ]          [ Refresh Data ]
```

### 2. "Continues in/from" Sidebar Line — Details.cshtml and Manage.cshtml

**Location:** Inside the existing "Quest Summary" `.card.modern-card` in the `col-md-4` sidebar — appended after the last `<p>` before the `<hr>` that precedes Quick Stats.

**Exact markup pattern (on the original quest — "Continues in"):**
```html
@if (Model.FollowUpQuest != null)
{
    <p>
        <i class="fas fa-scroll text-warning me-2"></i>
        <strong>Continues in:</strong>
        <a href="@Url.Action("Details", "Quest", new { id = Model.FollowUpQuest.Id })">
            @Model.FollowUpQuest.Title
        </a>
    </p>
}
```

**Exact markup pattern (on the follow-up quest — "Continues from"):**
```html
@if (Model.OriginalQuest != null)
{
    <p>
        <i class="fas fa-scroll text-warning me-2"></i>
        <strong>Continues from:</strong>
        <a href="@Url.Action("Details", "Quest", new { id = Model.OriginalQuest.Id })">
            @Model.OriginalQuest.Title
        </a>
    </p>
}
```

**Icon:** `fa-scroll` with `text-warning` (`#ffc107`), consistent with all other meta icons on the page.

**Source:** CONTEXT.md D-08, D-09 — these are locked decisions, not discretionary.

### 3. Follow-Up Creation Form — /Quest/CreateFollowUp/{id}

**Layout:** Reuse the exact `Create.cshtml` two-column layout (`col-lg-8 col-md-7` main form + `col-lg-4 col-md-5` sidebar).

**Card structure:** Use `.card.modern-card` + `.card-header.modern-card-header` + `.card-body.modern-card-body` — identical to Create.cshtml.

**Header:**
```html
<div class="card-header modern-card-header">
    <h2 class="mb-0">
        <i class="fas fa-scroll text-warning me-2"></i>
        Create Follow-Up Quest
    </h2>
</div>
```

**Pre-filled fields (from D-01, D-02, D-03, D-04):**
- Title: pre-filled with `{original.Title} - Part 2`, fully editable
- Description: pre-filled from original, fully editable
- ChallengeRating: pre-filled from original, fully editable
- TotalPlayerCount: pre-filled from original, fully editable
- DungeonMasterId: pre-filled from original, fully editable
- DungeonMasterSession: always `false` (not copied)
- ProposedDates: always empty — DM must add at least one date before saving

**Info banner above the dates section:**
```html
<div class="alert alert-info mb-3">
    <i class="fas fa-info-circle me-2"></i>
    This form is pre-filled from the original quest. Update the title, add at least one date, and save to create the follow-up.
</div>
```

**Validation behaviour:**
- Submitting with zero proposed dates must fail with a server-side validation error (existing `asp-validation-summary="All"` pattern)
- Error message: "At least one proposed date is required before saving a follow-up quest."

**Button row (bottom of form):**
```html
<hr>
<div class="d-flex justify-content-between">
    <a href="@Url.Action("Manage", "Quest", new { id = Model.OriginalQuestId })"
       class="btn btn-secondary">
        <i class="fas fa-arrow-left me-2"></i>
        Cancel
    </a>
    <button type="submit" class="btn btn-success">
        <i class="fas fa-scroll me-2"></i>
        Save Follow-Up Quest
    </button>
</div>
```

**Sidebar (right column):** Render an informational card summarising what gets pre-imported:
```html
<div class="card modern-card mb-3">
    <div class="card-header modern-card-header">
        <h5>Pre-Approved Players</h5>
    </div>
    <div class="card-body modern-card-body">
        <p class="text-muted">
            <i class="fas fa-users me-2"></i>
            Selected players from the original quest will be automatically added as pre-approved signups once this quest is saved.
        </p>
        <ul class="list-unstyled small">
            @foreach (var player in ViewBag.PreApprovedPlayers)
            {
                <li><i class="fas fa-check-circle text-success me-2"></i>@player.Name</li>
            }
        </ul>
    </div>
</div>
```

## Interaction Contracts

### "Create Follow-Up Quest" button
- **Trigger:** GET request to `/Quest/CreateFollowUp/{id}`
- **Guard:** `[Authorize(Policy = "DungeonMasterOnly")]` + controller checks that DM is owner or user is admin (same pattern as `Manage` action)
- **Guard 2:** If `FollowUpQuestId` already set on entity, redirect back to Manage with error TempData: "A follow-up quest already exists for this quest."
- **No confirmation modal** — creating a follow-up is not destructive; the DM is taken to a form, not committed immediately

### "Save Follow-Up Quest" form submit
- **Trigger:** POST to `/Quest/CreateFollowUp/{id}`
- **Success:** Redirect to `Manage` page of the newly created follow-up quest
- **Failure (validation):** Re-render the form with validation summary intact (existing `asp-validation-summary="All"` pattern)

### "Continues in/from" link
- **Trigger:** Standard `<a href>` navigation — no JavaScript required
- **Destination:** `Details.cshtml` of the linked quest
- **Visible to:** All users who can view the quest (no auth guard on the sidebar line itself; the link target is already protected by the Details action's own auth)

### Page auto-refresh
- No change to the existing 30-second auto-refresh behaviour on Details.cshtml
