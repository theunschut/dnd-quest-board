## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage                                              |
|-------|-------|----------------------------------------------------|
| xs    | 4px   | Badge padding, icon gaps, rarity badge `2px 6px`   |
| sm    | 8px   | Card padding (compact cards), gap between buttons   |
| md    | 16px  | Glass card padding, form-container padding, section gaps |
| lg    | 24px  | Between major card blocks on a page                |
| xl    | 32px  | Page top/bottom breathing room                     |
| 2xl   | 48px  | Not used in mobile views — desktop-only            |
| 3xl   | 64px  | Not used in mobile views — desktop-only            |

Exceptions:
- Touch targets: All `.btn`, `input`, `select`, `textarea`, `.form-control`, `.form-select` have `min-height: 44px` — declared in `mobile.css` (INFRA-06). Do NOT redefine in per-page CSS.
- Icon-only action buttons in ShopManagement Index: `btn-sm` with `padding: 4px 8px` override to fit multiple buttons horizontally without wrapping on 320px screens.
- Checkbox and radio inputs: exempt from 44px rule (Phase 18 fix — `23e378f`). Per `mobile.css` selector: `input:not([type=checkbox]):not([type=radio])`.

Source: `mobile.css` (INFRA-06), 19-CONTEXT.md D-01 / D-08, Phase 18 commit `23e378f`

## Typography

| Role    | Size      | Weight       | Line Height | Notes                              |
|---------|-----------|--------------|-------------|------------------------------------|
| Body    | 16px      | 400 (regular) | 1.5        | Set in `mobile.css` `body` rule. Prevents iOS auto-zoom. |
| Label   | 14px      | 700 (bold)   | 1.4         | Form labels, card meta lines, `.form-label` — visually distinct from body text |
| Heading | 20px      | 700 (bold)   | 1.2         | Glass card `h5` headings            |
| Display | 24px      | 700 (bold)   | 1.2         | Page-level `h2` (e.g. "Shop Management") |

Declared weights: 400 (regular) + 700 (bold). No third weight.

Parchment heading text treatment (applied to glass card h5 and display headings inside glass cards):
- `color: #F4E4BC !important`
- `text-shadow: 2px 2px 4px rgba(0,0,0,0.9), -1px -1px 2px rgba(0,0,0,0.9)`

Subdued / meta text (small labels, form-text, price hints):
- `color: rgba(244, 228, 188, 0.7) !important`
- `text-shadow: 1px 1px 3px rgba(0, 0, 0, 0.9) !important`
- `font-size: 0.875rem` (14px)

Source: `mobile.css`, `dm-editprofile.mobile.css`, `quest-edit.mobile.css`, 19-CONTEXT.md

### Admin Users Card (admin-users.mobile.css)

- Action row: `d-flex flex-wrap gap-2` — all applicable buttons left-to-right; conditionals (Promote/Demote) apply same logic as desktop
- Button styles: Promote to Admin (`btn-warning`), Promote to DM (`btn-info`), Demote (`btn-secondary`), Edit (`btn-primary btn-sm`), Delete (`btn-danger btn-sm`)

### Admin Quests Card (admin-quests.mobile.css)
- Glass card wrapping each quest
- Top row: `fa-dragon text-muted me-2` + quest title (parchment)
- Meta line: `<small class="text-muted">DM: @quest.DungeonMaster?.Name</small>`
- Status badge inline with title (same badge logic as desktop: bg-dark/bg-primary/bg-success)
- Action row: Edit (`btn-primary btn-sm`) + Delete (`btn-danger btn-sm`)

### Admin Form Card — EditUser + ResetPassword (admin-form.mobile.css)
- Shared single CSS file (Claude's discretion, 19-CONTEXT.md)
- Glass card full-width single-column

- EditUser: Name, Email, HasKey checkbox, Reset Password link, Save Changes button
- ResetPassword: `alert-warning` kept (functional context, D-06), New Password, Confirm, submit button

### ShopManagement Index Item Card (shop-management-index.mobile.css)
- Flat single-column list (no section separators) — all items combined, ordered: Pending Review first, then alphabetical by name (Claude's discretion, D-07)
- Glass card per item
- Card structure: rarity badge (top-left) + item name (bold, parchment) + price (`X gp`, subdued) + status badge — all in one row/block
- Action buttons: icon-only `btn-sm` row — View (`fa-eye`), Edit (`fa-edit`), Archive/Reopen (`fa-archive`/`fa-box-open`), Deny (`fa-times-circle btn-warning`), Delete (`fa-trash btn-danger`)
  - Each button requires BOTH `title="Action name"` (hover tooltip) AND `aria-label="Action name"` (screen reader). Exact labels: `aria-label="View item"`, `aria-label="Edit item"`, `aria-label="Archive item"` / `aria-label="Reopen item"`, `aria-label="Deny item"`, `aria-label="Delete item"`
- Authorization guards (AdminOnly for delete/publish) preserved from desktop
- "Add New Item" button: full-width `btn-primary` at top, `fa-plus me-2`
- "View Shop" link: full-width `btn-outline-secondary` below Add button
- Empty state: glass card, centered, `fa-box-open fa-3x` icon + heading "No items yet" + body text "Add the first item to your shop."
- Deny Item modal: Bootstrap 5 modal (copy from desktop Index — works on mobile)
- Bulk Actions modal: Bootstrap 5 modal stub (copy from desktop Index — shows placeholder)

### ShopManagement Create / Edit Forms (shop-management-create.mobile.css / shop-management-edit.mobile.css)
- Single-column glass card form — all fields `col-12`
- Fields: Name, Type (select), Description (textarea), Rarity (select), Quantity (number), Price (number, full-width, NO input-group buttons), ReferenceUrl, AvailabilityFrom, AvailabilityUntil
- Price field: full-width `<input class="form-control" />` with no input-group wrapper; omit dice and calculator buttons (D-12)
- Price suggestion hint: `<div id="price-suggestion" class="form-text">` — shown below price field, updated by `updatePriceSuggestion()` on rarity change
- Availability window toggle: `toggleAvailabilityWindow()` function kept; date inputs visible/hidden based on checkbox
- Rarity CSS animation: `@keyframes legendary-glow` kept in `@section Scripts` `<style>` block (rarity badge visual)
- Edit view only: `alert-info` status info alert kept (functional context, D-15)
- Button row: `d-flex justify-content-between` — Cancel (left, `btn-secondary`) + Save/Create (right, `btn-primary`)

### Shop Details Card (shop-details.mobile.css)
- Content inlined (no `_ShopItemDetailsContent` partial), non-modal path only — `@if (isModal)` branch skipped entirely (D-17, D-18)
- Glass card header: item name (parchment h5) + type badge inline
- Glass card body: rarity badge (`.rarity-{rarity}` class) + description + price (`X gp`, bold) + purchase button (full-width `btn-primary`, `fa-shopping-cart me-2`)
- Toast notifications: TempData["Success"], TempData["Error"], TempData["GoldReceived"] — kept at top of page (D-19)
- Toast container: `position: fixed; top: 16px; right: 16px; z-index: 1055`

## Interaction Contract

### Navigation & Tap Targets
- All `.btn` elements meet 44px `min-height` — enforced by `mobile.css` (INFRA-06)
- Admin Users list rows: NOT tap-navigable (action buttons are the only interaction entry points)
- Admin Quests list rows: NOT tap-navigable (action buttons only)
- ShopManagement item cards: NOT tap-navigable; View button (`fa-eye`) is the explicit navigation affordance
- Shop Details: no row-level navigation (single item view)

### Button Layout
Per CLAUDE.md: `d-flex justify-content-between` — secondary/cancel on left, primary/submit on right.
Applied to: EditUser, ResetPassword, ShopMgmt Create, ShopMgmt Edit.

Action button rows on list cards (Admin Users, Admin Quests, ShopManagement items): `d-flex flex-wrap gap-2` — wrap on narrow screens.

Icon-only buttons (ShopManagement item cards):
- Use `btn-sm` with BOTH `title="Action name"` (hover tooltip) AND `aria-label="Action name"` (screen reader) on every button
- Exact `aria-label` values: `"View item"`, `"Edit item"`, `"Archive item"` / `"Reopen item"`, `"Deny item"`, `"Delete item"`
- Size: `padding: 4px 8px` override — prevents wrapping of 4–5 buttons on 320px screens
- All icons: `fas` prefix, no text label

### Form Interactions
- `toggleAvailabilityWindow()` — shows/hides AvailabilityFrom + AvailabilityUntil date inputs on checkbox change (ShopMgmt Create/Edit)
- `updatePriceSuggestion()` — fires on `#Rarity` change, updates `#price-suggestion` div text (ShopMgmt Create/Edit)
- `setRandomPrice()` and `setCalculatedPrice()` — omitted entirely (buttons removed per D-12)
- Rarity CSS animation (`@keyframes legendary-glow`) — kept in `<style>` inside `@section Scripts` on ShopMgmt Create/Edit
- `deleteUser()` / `deleteQuest()` / delete shop item JS — copied verbatim from desktop into `@section Scripts`
- Antiforgery: `@inject IAntiforgery Antiforgery` already global in `_ViewImports.cshtml` — do NOT re-inject. `@{ var tokens = Antiforgery.GetAndStoreTokens(ViewContext.HttpContext); }` block copied into each view that uses inline DELETE fetch.

### Modals
- "Deny Item" modal: Bootstrap 5 `data-bs-toggle="modal"` — copy from desktop verbatim; no JS changes needed
- "Bulk Actions" modal: Bootstrap 5 modal stub — copy from desktop verbatim; shows placeholder text

### Toast Notifications (Shop Details)
- TempData["Success"], TempData["Error"], TempData["GoldReceived"] — render at top of page using Bootstrap toast pattern
- Copy toast markup from any prior phase that uses TempData toasts (e.g. Phase 16 Shop/Index.Mobile.cshtml)
