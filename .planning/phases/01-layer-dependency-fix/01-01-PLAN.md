**Step 1: Refactor BaseRepository.cs to dual-generic pattern.**

Replace the current `BaseRepository<T>` with a dual-generic `BaseRepository<TModel, TEntity>` that:
- Accepts `QuestBoardContext dbContext` and `IMapper mapper` via primary constructor
- Has constraints: `where TModel : class, IModel` and `where TEntity : class, IEntity`
- Implements `IBaseRepository<TEntity>` (STILL entity-typed for now -- interfaces move to Domain in Plan 02)
- All CRUD methods continue to accept/return `TEntity` for now (the interface hasn't changed yet)
- Exposes `protected IMapper Mapper { get; }` for use by concrete repositories
- The `DbSet<TEntity>` and `DbContext` properties remain as-is

The new signature:
```csharp
using AutoMapper;
using EuphoriaInn.Repository.Entities;
using EuphoriaInn.Repository.Interfaces;
using EuphoriaInn.Domain.Models;
using Microsoft.EntityFrameworkCore;

namespace EuphoriaInn.Repository;

internal abstract class BaseRepository<TModel, TEntity>(QuestBoardContext dbContext, IMapper mapper) 
    : IBaseRepository<TEntity> 
    where TModel : class, IModel
    where TEntity : class, IEntity
{
    protected QuestBoardContext DbContext { get; } = dbContext;
    protected DbSet<TEntity> DbSet { get; } = dbContext.Set<TEntity>();
    protected IMapper Mapper { get; } = mapper;

    // All existing CRUD methods stay the same (entity-typed) for now
    // They will be changed to domain-model-typed in Plan 02 when interfaces move
}
```

**Step 2: Update all concrete repository constructors to pass IMapper.**

3. Add `using AutoMapper;` and `using EuphoriaInn.Domain.Models;` (or the specific model namespace) to using directives
