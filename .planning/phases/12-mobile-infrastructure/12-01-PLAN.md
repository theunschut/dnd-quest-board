<objective>
Build the mobile-detection backend: a request middleware that flags mobile User-Agents into `HttpContext.Items["IsMobile"]`, and an `IViewLocationExpander` that — on mobile requests — makes the Razor view engine look for `ViewName.Mobile.cshtml` before `ViewName.cshtml`. Register both in `Program.cs` without moving `UseSession()`. Also create the Wave 0 unit test files that lock in the detection and cache-key behavior.

Purpose: This is the atomic prerequisite for the entire Milestone 3 mobile build. Phases 13-16 add `.Mobile.cshtml` views; none of them resolve until this expander and middleware are live. INFRA-03's cache-key correctness (detection in `PopulateValues`, not `ExpandViewLocations`) is the single most failure-prone requirement and is locked by a dedicated test here.

Output: `MobileDetectionMiddleware.cs`, `MobileViewLocationExpander.cs`, two `Program.cs` additions plus three `using` directives, and two test files under `EuphoriaInn.IntegrationTests/Mobile/`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → server (User-Agent header) | Untrusted UA string crosses into `MobileDetectionMiddleware`; used only for layout selection |
| middleware → view engine | `HttpContext.Items["IsMobile"]` flag passes to the expander and `_ViewStart.cshtml` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | Spoofing | `MobileDetectionMiddleware` (User-Agent) | accept | UA is attacker-controllable but only selects a layout — mobile views serve the same data and same authorization as desktop. No access-control decision reads `IsMobile`. Spoofing a UA yields nothing beyond a different CSS/markup shell. ASVS L1: not security-relevant. |
| T-12-02 | Tampering | `HttpContext.Items["IsMobile"]` | accept | Boolean flag is server-set per request, not persisted, not user-writable across the boundary. No tampering vector beyond the UA header already covered by T-12-01. |
| T-12-03 | Information Disclosure | `MobileViewLocationExpander` path expansion | accept | Expander only rewrites view-file paths within the app's own Views tree; it cannot reference paths outside the configured view locations. No traversal input is user-controlled. |
| T-12-SC | Tampering | npm/nuget installs | mitigate | No packages installed this phase (RESEARCH.md Package Legitimacy Audit: zero new dependencies). No [ASSUMED]/[SUS] packages — no install checkpoint required. |
