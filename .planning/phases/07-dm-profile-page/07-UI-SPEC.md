# Phase 7 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Bootstrap 5.3.0 via CDN) |
| Preset | not applicable |
| Component library | Bootstrap 5 — existing project-wide framework |
| Icon library | Font Awesome 6.4.0 (CDN) |
| Font | Cinzel (Google Fonts, display/headings only); system-ui stack for body |

**Note:** This is a server-rendered ASP.NET Core 8 MVC application. No shadcn/React. All styling is Bootstrap 5 utility classes plus custom CSS classes already defined in `site.css`. The `modern-card` / `modern-card-header` / `modern-card-body` pattern is the established project standard for all new views and MUST be used on every new surface in this phase.

---

## Spacing Scale

Bootstrap utility classes map to the 8-point scale. Use these classes exclusively — no inline px values.

| Token | Value | Bootstrap Class | Usage |
|-------|-------|-----------------|-------|
| xs | 4px | `p-1` / `m-1` / `gap-1` | Icon gaps, inline badge padding |
| sm | 8px | `p-2` / `m-2` / `gap-2` | Compact element spacing, icon `me-2` |
| md | 16px | `p-3` / `m-3` / `gap-3` | Default element spacing, card body padding |
| lg | 24px | `p-4` / `m-4` / `gap-4` | Section padding within cards |
| xl | 32px | `p-5` / `mb-4` | Layout gaps between sections |
| 2xl | 48px | `py-5` | Empty-state vertical centering |
| 3xl | 64px | n/a | Not used in this phase |

Exceptions:
- Profile photo container: fixed 128px × 128px (rounded-circle). Not a spacing token — it is a component dimension.
- Quest list rows: `py-2` (8px vertical) per row to match existing table cell density.
- Icon-only touch targets in the DM directory link: minimum 44×44px tap area ensured by `p-2` on the anchor wrapping the icon.

## New Views and Their Layouts

This section is prescriptive markup guidance consumed by the executor.

### 1. `/DungeonMaster/Profile/{id}` — Public DM Profile Page

Two-column Bootstrap grid: `col-lg-4` (photo + bio sidebar) | `col-lg-8` (quest history list).

**Left column (col-lg-4):**
```
card.modern-card
  card-body.modern-card-body.text-center
    IF photo exists:
      img.rounded-circle (128x128, object-fit: cover)
        src="Url.Action('GetDMProfilePicture', new { id = Model.DungeonMasterId })"
    ELSE:
      div.dm-profile-placeholder
        i.fas.fa-user.fa-5x.text-muted
    h2.mt-3  [DM Name]
    p.text-muted
      i.fas.fa-crown.me-1  "Dungeon Master"

  IF isOwner OR isAdmin:
    hr
    a.btn.btn-warning.w-100
      i.fas.fa-edit.me-2  "Edit Profile"
      href="/DungeonMaster/EditProfile/{id}"

card.modern-card.mt-3
  card-header.modern-card-header
    h5.mb-0
      i.fas.fa-scroll.me-2  "About"
  card-body.modern-card-body
    IF bio is not null/empty:
      p [bio text, white-space: pre-wrap]
    ELSE:
      p.text-muted.fst-italic  "No bio provided yet."
```

**Right column (col-lg-8):**
```
card.modern-card
  card-header.modern-card-header
    h2.mb-0
      i.fas.fa-list.me-2  "Quest History"
  card-body.modern-card-body
    IF quests.Any():
      table.table.table-striped.table-hover
        thead: tr > th[Title] th[Date] th[Difficulty]
        tbody: foreach quest (ordered date desc):
          tr
            td > a[quest.Title] href="/Quest/Details/{quest.Id}"
            td [quest.FinalizedDate?.ToString("MMM dd, yyyy")]
            td > span.badge [difficulty color from existing difficulty badge pattern]
    ELSE:
      div.text-center.py-5
        i.fas.fa-dragon.fa-3x.text-warning.mb-3
        h4  "No Quests Yet"
        p.text-muted  "This Dungeon Master hasn't run any quests yet."
```

Difficulty badge colors reuse the existing site pattern:
- Easy: `bg-success`
- Medium: `bg-warning text-dark`
- Hard: `bg-danger`
- Deadly: `bg-purple` (custom: `background-color: #6f42c1; color: white`)

### 2. `/DungeonMaster/EditProfile` and `/DungeonMaster/EditProfile/{id}` — Edit Form

Single-column layout, `col-lg-8 mx-auto` centered card.

```
card.modern-card
  card-header.modern-card-header
    h2.mb-0
      i.fas.fa-user-edit.me-2  "Edit DM Profile"

  card-body.modern-card-body
    form [method=post, enctype=multipart/form-data]
      hidden [asp-for=DungeonMasterId]
      validation-summary

      div.row
        div.col-md-4.mb-3   ← photo column
          label.form-label  "Profile Photo"
          IF current photo exists:
            div.mb-2
              img.rounded-circle (128x128)
                src="Url.Action('GetDMProfilePicture', ...)"
                alt="Current profile photo"
          input[type=file, asp-for=ProfilePictureFile, class=form-control, accept="image/*"]
          small.form-text.text-muted  "JPG, PNG or GIF — max 5 MB"
          span.text-danger [asp-validation-for=ProfilePictureFile]
          div#dmFileSizeError.text-danger [hidden by default]

        div.col-md-8   ← bio column
          div.mb-3
            label.form-label  "Bio"
            textarea[asp-for=Bio, class=form-control, rows=6, maxlength=2000]
            span.text-danger [asp-validation-for=Bio]
            small.form-text.text-muted  "Introduce yourself to the players (max 2000 characters)."

      hr
      div.d-flex.justify-content-between
        a.btn.btn-secondary href="Url.Action('Profile', new { id = ... })"
          i.fas.fa-arrow-left.me-2  "Cancel"
        button[type=submit].btn.btn-warning
          i.fas.fa-save.me-2  "Save Profile"
```

## Interaction Contract

### Edit Profile form — client-side file validation

Mirrors the pattern in `GuildMembers/Edit.cshtml` exactly:

- On `change` of the file input: check `file.size > 5 * 1024 * 1024` → show `#dmFileSizeError`, clear input.
- On `change` of the file input: check `!['image/jpeg','image/jpg','image/png','image/gif'].includes(file.type)` → show `#dmFileSizeError`, clear input.
- On valid file: hide `#dmFileSizeError`.

No other JavaScript is needed on any new view in this phase.

### Profile page — "Edit Profile" button visibility

Server-side Razor condition: visible only when `User.Identity.IsAuthenticated && (isOwnerDM || User.IsInRole("Admin"))`.
- `isOwnerDM` = current logged-in user's `Id` equals the profile's `DungeonMasterId`.
- Not visible to anonymous or to authenticated non-DM/non-admin users.

### DM directory link

The DM name in `Players/Index.cshtml` becomes a hyperlink with no new hover effect beyond the existing table hover rule already in `site.css` (`.modern-card .table-hover > tbody > tr:hover > td`).
