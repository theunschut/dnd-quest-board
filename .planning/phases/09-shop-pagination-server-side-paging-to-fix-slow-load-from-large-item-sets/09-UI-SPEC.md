## Components

### 1. Server-Side Search Input (replaces JS search bar)

Location in view: same position as the existing `.shop-search` div (between filter row and inventory grid).

Markup pattern:
```html
<div class="shop-search mb-3">
  <div class="input-group justify-content-center" style="max-width: 700px; margin: 0 auto;">
    <input type="text" name="search" class="form-control"
           placeholder="Search items by name or description..."
           value="@Model.SearchQuery" autocomplete="off" form="shop-filter-form" />
    <button type="submit" class="btn filter-apply-btn" form="shop-filter-form">
      <i class="fas fa-search me-1"></i>Search
    </button>
  </div>
</div>
```

- The search input submits into the existing filter form (via `form="shop-filter-form"`), or the filter form wraps it directly.
- Removing the text and submitting clears the search and resets to page 1.
- The "Search by description" checkbox is REMOVED along with the JS function. Server-side search matches name AND description by default.
- Input focus ring: `border-color: #ffc107; box-shadow: 0 0 0 0.2rem rgba(255,193,7,0.25)` — matching `.filter-sort-select:focus`.

Source for max-width and centering: existing `.shop-search` div in `Index.cshtml` line 240.

### 2. Pagination Component

Location in view: immediately after the closing tag of `.shop-inventory`, before the floating merchant button.

Rendered only when `Model.TotalPages > 1`.

Markup pattern:
```html
<nav aria-label="Shop page navigation" class="shop-pagination mt-3 mb-4 text-center">
  <ul class="pagination justify-content-center">
    <li class="page-item @(Model.CurrentPage <= 1 ? "disabled" : "")">
      <a class="page-link shop-page-link" href="@BuildPageUrl(Model.CurrentPage - 1)"
         aria-label="Previous page">
        <i class="fas fa-chevron-left me-1"></i>Previous
      </a>
    </li>

    @* Page number links — see page window rules below *@

    <li class="page-item @(Model.CurrentPage >= Model.TotalPages ? "disabled" : "")">
      <a class="page-link shop-page-link" href="@BuildPageUrl(Model.CurrentPage + 1)"
         aria-label="Next page">
        Next<i class="fas fa-chevron-right ms-1"></i>
      </a>
    </li>
  </ul>
  <p class="shop-pagination-count text-muted mb-0">
    Showing @((Model.CurrentPage - 1) * 12 + 1)–@(Math.Min(Model.CurrentPage * 12, Model.TotalItems))
    of @Model.TotalItems items
  </p>
</nav>
```

Page window rules (how many number links to show):
- Always show first page (page 1) and last page (`Model.TotalPages`).
- Show current page and ±2 pages from current.
- Use an ellipsis element `<li class="page-item disabled"><span class="page-link">…</span></li>` when there is a gap between page 1 and the window start, or between the window end and last page.
- Example at page 5 of 10: `1 … 3 4 [5] 6 7 … 10`.
- Example at page 1 of 5: `[1] 2 3 4 5` (no ellipsis needed).

Active page link: `class="page-item active"` on `<li>`, rendered as `<span class="page-link">` (not an anchor) to prevent re-navigation to current page.

### 3. Filter Form Extension (search + page reset)

The existing filter form (`<form method="get" ...>`) gains:
- A hidden `<input type="hidden" name="page" value="1" />` — ensures any filter/sort change resets to page 1 (D-07).
- The search text input (see component 1 above), placed inside the form.
- The existing `BuildTabUrl` helper in `@functions` is extended to accept `string? search` and always emit `page=1` (omitted when value is 1 to keep URLs clean, or included explicitly — planner decides).


Text pattern: "Showing 1–12 of 47 items" (source: D-01, 12 items per page).

## Interaction Contract

| Trigger | Behavior |
|---------|----------|
| User types in search input and submits filter form | Full page navigation to `?search={value}&page=1` + all active filters/sort carry forward |
| User clicks a pager page number | Full page navigation to `?page={n}` + all active filters/sort/search carry forward |
| User clicks Previous | Navigate to `?page={current-1}` + all state |
| User clicks Next | Navigate to `?page={current+1}` + all state |
| User changes rarity checkbox or sort and clicks Apply Filters | Resets page to 1; search carries forward |
| User clicks a category tab (All / Equipment / Magic Items / Quest Items) | Resets page to 1; search and sort carry forward |
| User clears search (empty input + submit) | Resets page to 1; filters/sort carry forward |
| Previous button at page 1 | Rendered as disabled; no navigation |
| Next button at last page | Rendered as disabled; no navigation |
| JavaScript disabled | All interactions work identically — no JS dependency (D-04 confirmed: `filterShopItems` removed) |

Source: CONTEXT.md decisions D-03, D-07, D-08.
