## Spacing Scale

Declared values (must be multiples of 4). Expressed as Bootstrap spacing utilities and raw px:

| Token | Value | Bootstrap Util | Usage |
|-------|-------|---------------|-------|
| xs | 4px | `gap-1`, `me-1`, `ms-1` | Icon-to-text gaps, badge internal padding |
| sm | 8px | `p-2`, `py-2`, `gap-2`, `m-2` | Card internal padding, badge margin from card edge |
| md | 16px | `p-3`, `px-3`, `mb-3`, `gap-3` | Card body padding, between-section gaps |
| lg | 24px | `p-4`, `mb-4`, `gap-4` | Card stack gap, section breaks within Details view |
| xl | 32px | `mb-5` | Page-level top padding above card list |
| 2xl | 48px | — | Not used in mobile views |
| 3xl | 64px | — | Not used in mobile views |

Exceptions:
- Touch targets: 44px minimum height on all interactive elements (`.btn`, inputs, `.nav-link`) — set globally in `mobile.css`. Vote buttons inherit this without per-page override.
- Signed-up badge: `position: absolute; top: 8px; right: 8px` (Bootstrap `top-0 end-0 m-2`) — 8px inset from card edge.
- Participant row: `padding: 8px 0` (`py-2`) with `border-bottom` separator.

Source: `mobile.css` INFRA-06; D-01 / D-02 locked decisions.

## Typography

Four sizes, two weights only.

| Role | Size | Weight | Line Height | Font | Bootstrap Class |
|------|------|--------|-------------|------|----------------|
| Body | 16px | 400 | 1.5 | System sans-serif | — (mobile.css baseline) |
| Label / meta | ~14px (0.875em) | 400 | 1.4 | System sans-serif | `<small>` or `small` class |
| Card title / section heading | 20px | 700 | 1.3–1.2 | System sans-serif (card) / Cinzel (section) | `h5` or `fw-bold fs-5` |
| Page heading | 24px | 700 | 1.2 | Cinzel, serif | `h4` + Cinzel via per-page CSS |

Notes:
- Body 16px is declared in `mobile.css` — prevents iOS auto-zoom on input focus. Do not redeclare in per-page CSS.
- Meta text uses Bootstrap's `<small>` element (0.875em × 16px ≈ 14px). Do not declare a fixed 12px size in per-page CSS.
- Cinzel applies only to page headings and the Quest Log section header. Card titles use system sans-serif at 20px.
- Two weights only: 400 (body, label, meta) and 700 (all headings, card titles, section headings). No intermediate 600 weight.

Source: `mobile.css` line 14; `_Layout.Mobile.cshtml` Google Fonts CDN line 10; CONTEXT.md Claude's Discretion.

## Color

The mobile views use Bootstrap 5 contextual color tokens exclusively. No hex values are introduced in per-page CSS; custom hex is only in `mobile.css` (already set).

| Role | Bootstrap Token | Approximate Hex | Usage |
|------|----------------|----------------|-------|
| Dominant (60%) | `bg-dark` / `#212529` | #212529 | Page background (set by `.mobile-layout` in `mobile.css`) |
| Secondary (30%) | `bg-dark` + `border` | #343a40 | Quest cards, participant rows, list items |
| Accent (10%) | `text-primary` / Bootstrap blue | #0d6efd | Finalized status badge (`bg-primary`); primary CTAs |
| Success | `bg-success` / Bootstrap green | #198754 | Open status badge; signed-up indicator badge |
| Destructive | `bg-danger` / Bootstrap red | #dc3545 | Revoke signup button only |
| Muted | `text-muted` | #6c757d | Meta text (DM name, date, character name) |

### Status Badge Color Contract (D-05 — locked)

| Quest State | Badge Class | Icon Class | Label Text |
|-------------|------------|-----------|-----------|
| Open (not finalized) | `bg-success` | `fas fa-clock` | "Open" |
| Finalized (future date or no date) | `bg-primary` | `fas fa-check-circle` | "Finalized" |
| Done (finalized AND date is yesterday or earlier) | `bg-secondary` | `fas fa-flag-checkered` | "Done" |

Three-way status check logic (from RESEARCH.md, verified against Details.cshtml):
```
IsFinalized AND FinalizedDate.Value.Date <= DateTime.UtcNow.AddDays(-1).Date → Done (bg-secondary)
IsFinalized (no date, or future date)                                         → Finalized (bg-primary)
Neither                                                                        → Open (bg-success)
```

Accent reserved for: Finalized status badge, primary action buttons (Vote Yes, Sign up). Not applied to card backgrounds, meta text, or informational icons.

Source: D-05 locked decision; RESEARCH.md Data Binding Inventory — Status badge logic; CONTEXT.md specifics.

### View 2: Quest/Details.Mobile.cshtml (QVIEW-01, QVIEW-02)

**Layout structure (single column, top to bottom):**
```
1. Quest title + CR badge (header row)
2. DM name + status badge (meta row)
3. Quest description (full text, pre-wrap)
4. [Voting section — when not finalized or user not signed up]
   4a. Proposed date list (via _Calendar partial or simplified list — see note)
   4b. Vote Yes / Vote No / Vote Maybe buttons (full-width, stacked, min-height 44px)
5. [Participant list — when finalized]
   5a. Section heading "Adventurers"
   5b. Stacked rows: player name + character name + role badge
6. [Sign-up / revoke controls]
```

**Voting buttons (QVIEW-01 — 44px enforced by mobile.css):**
```
<div class="d-grid gap-2">
  <button class="btn btn-success" onclick="changeVoteToYes(@Model.Quest?.Id)">
    <i class="fas fa-check me-2"></i>Vote Yes
  </button>
  <button class="btn btn-danger" onclick="changeVoteToNo(@Model.Quest?.Id)">
    <i class="fas fa-times me-2"></i>Vote No
  </button>
  <button class="btn btn-warning" onclick="changeVoteToMaybe(@Model.Quest?.Id)">
    <i class="fas fa-question me-2"></i>Vote Maybe
  </button>
</div>
```
- `d-grid gap-2` renders buttons full-width, stacked vertically
- `min-height: 44px` inherited from `mobile.css` `.btn` rule — no per-page override needed
- AJAX pattern: same fetch-based JS as desktop (`changeVoteToYes`, `changeVoteToNo`, `changeVoteToMaybe`)
- Antiforgery token: use globally-injected `Antiforgery.GetAndStoreTokens(ViewContext.HttpContext)` — no `@inject` directive in the mobile view
- No page refresh on button tap itself — only after fetch resolves

## Interaction Contract

### Tap Navigation

All tap targets use `onclick="window.location.href='...'"` — established codebase pattern. No JavaScript event listeners. Touch area must be the full card width and height.

### AJAX Voting (QVIEW-01)

- Voting buttons call `changeVoteToYes(questId)`, `changeVoteToNo(questId)`, `changeVoteToMaybe(questId)`
- Functions defined in the `@section Scripts` block of the mobile view (copied verbatim from desktop, minus the duplicate `@inject`)
- Each function POSTs to `/Quest/ChangeVoteToYes/{id}` (or No / Maybe) with `__RequestVerificationToken` in the form body
- On HTTP 200: `location.reload()` — full page refresh
- On error: `alert("Failed to change vote: " + text)`
- Antiforgery token rendered server-side: `'@tokens.RequestToken'` inside the `<script>` block
- No page refresh on button tap itself — only after fetch resolves

### Revoke Signup

- `revokeSignup(questId)` function in `@section Scripts`
- Sends DELETE to `/Quest/RevokeSignup/{id}` with antiforgery token
- Browser `confirm()` before sending — user must acknowledge
- On success: `location.reload()`

### No New JS Patterns

Phase 13 introduces no new JavaScript patterns. All JS is adapted from `Quest/Details.cshtml` lines 861–880 (verified in RESEARCH.md).

## CSS Architecture Contract

### File Map

| File | Scope | New/Existing |
|------|-------|-------------|
| `wwwroot/css/mobile.css` | Global mobile baseline — DO NOT MODIFY | Existing |
| `wwwroot/css/home.mobile.css` | Quest Board mobile only | New (Phase 13) |
| `wwwroot/css/quests.mobile.css` | Quest Details mobile only | New (Phase 13) |
| `wwwroot/css/quest-log.mobile.css` | Quest Log mobile only | New (Phase 13) |

### Loading Pattern

Each mobile view declares:
```cshtml
@section Styles {
    <link href="~/css/{page}.mobile.css" asp-append-version="true" rel="stylesheet" />
}
```

The `_Layout.Mobile.cshtml` renders `@await RenderSectionAsync("Styles", required: false)` at line 164 (after Bootstrap/FA CDN, before `</body>`). This is correct — modern browsers tolerate late-loaded stylesheets.

### CSS Authoring Rules

1. No `@media` queries — per-page CSS is mobile-only by definition
2. No `@import` — do not reference `site.css`, `quests.css`, or any desktop CSS
3. No hex values for theme colors — use `#343a40` and `#495057` for card surfaces (Bootstrap's `gray-800` / `gray-600` palette); `#212529` is set by `.mobile-layout` in `mobile.css`
4. Selectors must be page-scoped (prefix with `.home-page`, `.quest-details-page`, etc. where ambiguous — `ViewData["BodyClass"]` sets the body class)
