## Spacing Scale

Declared values (multiples of 4 only). Source: extracted from `mobile.css`, `dm-create.mobile.css`, `character-form.mobile.css`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps (`me-1`), small badge padding |
| sm | 8px | Item row bottom margin (`mb-2`), list entry separator padding |
| md | 16px | Glass card inner padding — all four views (`p-3` = 16px Bootstrap step) |
| lg | 24px | Section vertical spacing between cards |
| xl | 32px | Page-level top margin, form field group spacing via `mb-3` |

**Bootstrap utility mapping (in use across phases 12–17):**
- `mb-2` = 8px, `mb-3` = 16px, `px-2` = 8px, `gap-2` = 8px, `mt-2` = 8px
- `p-3` = 16px (used inside glass cards for body sections)

**Exceptions:**
- Touch targets: 44px minimum height — enforced by `mobile.css` on `.btn`, `input`, `select`, `textarea`, `.form-control`, `.form-select`. No exceptions in Phase 18.
- Recap textarea: `rows="6"` (reduced from desktop `rows="10"`) — textarea remains scrollable; exact height is at executor's discretion per CONTEXT.md Discretion section.
- Container horizontal padding: `px-2` (8px each side) — set globally by `mobile.css .container-fluid`.

**Glass card padding per view (all 16px — `md` token):**
- Quest Edit main form card: `padding: 16px`
- CreateFollowUp main form card: `padding: 16px`
- CreateFollowUp Pre-Approved Players card: `padding: 16px`
- DM EditProfile form card: `padding: 16px`
- QuestLog Details all cards: `padding: 16px`

## Typography

Source: `mobile.css` (baseline), `dm-profile.mobile.css`, `quest-log.mobile.css`, `character-form.mobile.css`.

| Role | Size | Weight | Line Height | Color |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | `#F4E4BC` (parchment) on glass cards; `#1a0f08` on notice board outside cards |
| Label / small meta | 14px (0.875rem) | 400 (regular) | 1.3 | `rgba(244, 228, 188, 0.7)` (faded parchment) |
| Card heading (h5/h6) | 20px (1.25rem) | 700 (bold) | 1.2 | `#F4E4BC` with text-shadow |
| Page/section heading (h4) | 24px (1.5rem) | 700 (bold) — Cinzel serif | 1.2 | `#F4E4BC` with text-shadow |

**Labels and meta text are distinguished from body by their 14px size and faded parchment color — a second weight is not needed.**

**Text shadow for headings and parchment labels:**
`2px 2px 4px rgba(0,0,0,0.9), -1px -1px 2px rgba(0,0,0,0.9)` — copy verbatim from existing files.

**Faded parchment text shadow (meta/small):**
`1px 1px 3px rgba(0,0,0,0.9)` — copy verbatim from existing files.

**iOS zoom prevention:** `font-size: 16px` on `body` is already set in `mobile.css`. All `input`, `select`, and `textarea` elements inherit this and must NOT be set below 16px via CSS.

## Color

Source: `mobile.css`, `dm-create.mobile.css`, `dm-profile.mobile.css`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Notice board background image (`/images/Notice Board-blank2.jpg`) | Page background — set globally by `mobile.css`; do not redefine in per-page CSS |
| Secondary (30%) | `rgba(255, 255, 255, 0.15)` with `backdrop-filter: blur(15px)` | Glass card containers — all four new views use glass cards |
| Accent (10%) | `#F4E4BC` (parchment gold) | Card headings, form labels, body text inside glass cards, nav brand, quest titles |
| Accent-warm | `#FFD700` (gold) / Bootstrap `text-warning` | Icon accents (fa-scroll, fa-crown, fa-save), Add Date / Update Quest button faces |
| Destructive | Bootstrap `btn-danger` (`#dc3545`) | Remove date buttons, file validation errors |
| Info semantic | Bootstrap `alert-info` | "Pre-filled from original quest" notice on CreateFollowUp (D-09) |
| Warning semantic | Bootstrap `alert-warning` | "Players have signed up" notice on Quest Edit (D-02) |
| Success semantic | Bootstrap `bg-success` badge | Building Access — key available (QuestLog Details D-17) |

**Accent reserved for:**
- Card headings (h5, h6) inside glass cards
- Form labels (`.form-label`) inside glass cards
- Quest/section titles
- Navbar brand text

**NOT accent:** Badge text shadows, form inputs (white/near-white background for readability), Bootstrap `.btn` text (white via `mobile.css .btn-primary`).

**Glass card exact values (copy verbatim — established across phases 13–17):**
```

border: 1px solid rgba(255, 255, 255, 0.3);
border-radius: 12px;
```

## Interaction Contracts

### All form views (Quest Edit, CreateFollowUp, DM EditProfile)

| Interaction | Behaviour |
|-------------|-----------|
| Tap any input field | Native iOS/Android keyboard appears; 16px font prevents iOS auto-zoom |
| Tap datetime-local input (CreateFollowUp) | Native mobile date-time picker |
| Tap "Add Another Date" / "Add Date" | Appends new date row via `addProposedDate()` (Quest Edit) or `addDate()` (CreateFollowUp) |
| Tap "Remove" on date row | Removes row; calls `removeProposedDate(this)` (Quest Edit) or `removeDate(this)` → `renumberDates()` (CreateFollowUp) |
| Tap "Submit" | Client-side validation via `_QuestFormScripts` partial (Quest Edit) or native HTML5 required (CreateFollowUp); server-side on post |
| Tap file input (DM EditProfile) | Native file picker; JS validates size (5MB max) and type (JPG/PNG/GIF) on change |
| Tap "Back" / "Back to Quest" / "Back to Profile" | Navigates to `Url.Action("Manage", "Quest", ...)` or `Url.Action("Profile", "DungeonMaster", ...)` as appropriate |

### QuestLog Details

| Interaction | Behaviour |
|-------------|-----------|
| Tap character avatar (`character-mini-avatar`) | Image load; `onerror` replaces with `fa-user` placeholder span |
| Tap "Back to Quest Log" | Navigates to `Url.Action("Index", "QuestLog")` |
| Tap "Manage Quest" | Navigates to `Url.Action("Manage", "Quest", new { id = Model.Quest.Id })` — only visible when `ViewBag.CanEditRecap == true` |
| Tap "Save Recap" | Form post to `UpdateRecap` action — only available to DM (CanEditRecap gate) |

## CSS Architecture

Source: CONTEXT.md D-20.

Four new CSS files — one per view. No @media queries in any file (files are exclusively loaded by their paired `.Mobile.cshtml` via `_Layout.Mobile.cshtml`).

| File | Loaded by | Base pattern |
|------|-----------|--------------|
| `quest-edit.mobile.css` | `Quest/Edit.Mobile.cshtml` | `dm-create.mobile.css` (form card, 16px padding) |
| `quest-followup.mobile.css` | `Quest/CreateFollowUp.Mobile.cshtml` | `dm-create.mobile.css` (form card, 16px padding) |
| `dm-editprofile.mobile.css` | `DungeonMaster/EditProfile.Mobile.cshtml` | `character-form.mobile.css` (16px padding, photo-at-top) |
| `quest-log-detail.mobile.css` | `QuestLog/Details.Mobile.cshtml` | `quest-log.mobile.css` (glass card, parchment text) |

**Do not redefine in per-page CSS:**
- `min-height: 44px` on interactive elements — already in `mobile.css`
- `font-size: 16px` on body — already in `mobile.css`
- `.btn-primary` gradient — already in `mobile.css`
- `.alert-danger`, `.alert-warning`, `.alert-info` — Bootstrap defaults + `mobile.css .alert-danger` override

**Class naming convention (per-page scoped):**
- Quest Edit card: `.quest-edit-card-mobile`
- CreateFollowUp main card: `.quest-followup-card-mobile`
- CreateFollowUp players card: `.quest-followup-players-card`
- DM EditProfile card: `.dm-editprofile-card-mobile`
- QuestLog Detail main card: `.quest-log-detail-main-card`
- QuestLog Detail actions card: `.quest-log-detail-actions-card`
- QuestLog Detail stats card: `.quest-log-detail-stats-card`

Exact class names are at executor's discretion — the above are recommendations. Source: CONTEXT.md Discretion.
