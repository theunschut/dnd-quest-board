### 3. Omphalos URL Field

```html
<div class="mb-3">
    <label asp-for="OmphalosUrl" class="form-label">Omphalos URL</label>
    <input asp-for="OmphalosUrl" class="form-control" type="url"
           placeholder="https://omphalos.example.com" />
    <div class="form-text">
        Base URL of the Omphalos application. Used for the navbar link and SSO redirect.
    </div>
    <span asp-validation-for="OmphalosUrl" class="text-danger"></span>
</div>
```

- `type="url"` — browser validates URL format natively; no custom regex required.
- Field is optional (may be blank); blank value + `IsEnabled = false` is valid initial state.
- Placeholder shows expected format; does not substitute for the hint text.

### 4. Shared Secret Field

```html
<div class="mb-3">
    <label asp-for="OmphalosSharedSecret" class="form-label">Shared Secret</label>
    <input asp-for="OmphalosSharedSecret" class="form-control" type="password"
           autocomplete="off" />
    <div class="form-text">
        HMAC-SHA256 signing key. Leave blank to keep the existing value.
    </div>
    <span asp-validation-for="OmphalosSharedSecret" class="text-danger"></span>
</div>
```

- `type="password"` — field renders masked (SETT-03). Browser does NOT pre-fill password inputs, so the field always loads empty regardless of stored value.
- `autocomplete="off"` — prevents browser password manager from offering to save this field.
- Hint text "Leave blank to keep the existing value." is mandatory (D-09, SETT-04). Must appear as `.form-text` directly below the input.
- Empty submission = preserve existing secret (service-layer concern; UI contract is the hint text only).

### 5. Integration Enabled Checkbox

```html
<div class="mb-3">
    <div class="form-check">
        <input asp-for="IsEnabled" class="form-check-input" type="checkbox" />
        <label asp-for="IsEnabled" class="form-check-label">
            <i class="fas fa-toggle-on text-success me-2"></i>
            Enable Omphalos integration
        </label>
    </div>
    <div class="form-text">
        When unchecked, all Omphalos links and buttons are hidden and the SSO redirect returns 404.
    </div>
    <span asp-validation-for="IsEnabled" class="text-danger"></span>
</div>
```

- Checkbox follows the `form-check` / `form-check-input` / `form-check-label` pattern from `EditUser.cshtml` lines 41–48.
- Icon `fa-toggle-on` color `text-success` — visual cue for the enabled state (icon is decorative only).
- Hint text explains the effect of unchecking (SETT-05) so admins understand scope before saving.

### 6. Validation Summary

```html
<div asp-validation-summary="ModelOnly" class="text-danger mb-3"></div>
```

Placed at the top of the form body, before the first field. Pattern from `EditUser.cshtml` line 23.

### 7. Feedback Messages (TempData)

Settings saves use Bootstrap in-card alerts, not toast notifications. This matches the `Account/Profile` and `Quest/Manage` pattern (dismissible alert inside the card body), which is more appropriate than the Shop toast for a settings page that redirects back to itself.

**Success:**
```html
@if (TempData["SuccessMessage"] != null)
{
    <div class="alert alert-success alert-dismissible fade show" role="alert">
        <i class="fas fa-check-circle me-2"></i>
        @TempData["SuccessMessage"]
        <button type="button" class="btn-close" data-bs-dismiss="alert" aria-label="Close"></button>
    </div>
}
```

**Error:**
```html
@if (TempData["ErrorMessage"] != null)
{
    <div class="alert alert-danger alert-dismissible fade show" role="alert">
        <i class="fas fa-exclamation-triangle me-2"></i>
        @TempData["ErrorMessage"]
        <button type="button" class="btn-close" data-bs-dismiss="alert" aria-label="Close"></button>
    </div>
}
```

Alerts are placed at the top of `modern-card-body`, above the validation summary and form fields, so they are immediately visible after the POST-Redirect-GET cycle.

### 8. Button Row

```html
<hr>
<div class="d-flex justify-content-between">
    <a asp-controller="Admin" asp-action="Users" class="btn btn-secondary">
        <i class="fas fa-arrow-left me-2"></i>
        Back to Admin
    </a>
    <button type="submit" class="btn btn-primary">
        <i class="fas fa-save me-2"></i>
        Save Settings
    </button>
</div>
```

- `<hr>` before buttons — matches `EditUser.cshtml` line 51 and CLAUDE.md convention.
- Left: secondary "Back to Admin" link — navigates to `Admin/Users` (the admin landing page).
- Right: primary "Save Settings" submit button.
- `btn-primary` renders as the blue gradient defined in `site.css` lines 1193–1201 (`background: linear-gradient(135deg, #3498db, #2980b9)`).

### 9. Admin Navbar Settings Link

Added to the Admin dropdown in `_Layout.cshtml`, after the existing "Quest Management" item, with a `<li><hr class="dropdown-divider"></li>` separator:

```html
<li><hr class="dropdown-divider"></li>
<li>
    <a class="dropdown-item" asp-controller="Admin" asp-action="Settings">
        <i class="fas fa-plug me-2"></i>Integration Settings
    </a>
</li>
```

- Divider separates entity management items (Users, Quest Management) from configuration items (Integration Settings) — a logical grouping.
- Icon `fa-plug` matches the card header icon for visual consistency.
- Label: "Integration Settings" — specific enough to communicate scope; consistent with page title.
- Placement: end of Admin dropdown. (Context D-01: "after Quest Management" confirmed as placement.)

**Source:** `_Layout.cshtml` lines 37–47; CONTEXT.md Claude's Discretion item on settings link placement.

## Interaction Contract

| Trigger | Behaviour |
|---------|-----------|
| GET `/Admin/Settings` | Load current settings from DB via `IAdminSettingService.GetSettingsAsync()`. URL field pre-populated; secret field always empty (type="password"); checkbox reflects stored `IsEnabled` value. |
| POST `/Admin/Settings` (secret field empty) | Service preserves existing `OmphalosSharedSecret`; does not overwrite with empty string (SETT-04 / D-08). |
| POST `/Admin/Settings` (valid) | PRG pattern: save, set `TempData["SuccessMessage"]`, redirect to GET. Success alert renders above form. |
| POST `/Admin/Settings` (invalid model state) | Return `View(model)` with validation summary. No TempData set. |
| Checkbox unchecked + saved | `IsEnabled = false` stored. All Omphalos UI elements hidden on next request (Phase 11 concern; no immediate UI change on this page). |
