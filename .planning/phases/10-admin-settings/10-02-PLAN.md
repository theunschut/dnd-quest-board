```csharp
[HttpGet]
public async Task<IActionResult> Settings()
{
    var settings = await adminSettingService.GetSettingsAsync();
    var model = new SettingsViewModel
    {
        OmphalosUrl = settings.OmphalosUrl,
        // DO NOT populate OmphalosSharedSecret — password fields must load empty (D-09)
        // Populating it would cause a blank string to overwrite the secret on unedited saves
        IsEnabled = settings.IsEnabled
    };
    return View(model);
}

[HttpPost]
[ValidateAntiForgeryToken]
public async Task<IActionResult> Settings(SettingsViewModel model)
{
    if (!ModelState.IsValid)
        return View(model);

    await adminSettingService.SaveSettingsAsync(
        model.OmphalosUrl,
        model.OmphalosSharedSecret,   // null/empty = preserve existing secret (D-08)
        model.IsEnabled);

    TempData["SuccessMessage"] = "Integration settings saved successfully.";
    return RedirectToAction(nameof(Settings));
}
```

Security notes:
- `[Authorize(Policy = "AdminOnly")]` is already at class level — the new actions inherit it automatically; do NOT add it per-action
- `[ValidateAntiForgeryToken]` is required on the POST action (matches every other POST in AdminController)
- OmphalosSharedSecret is intentionally not pre-populated on GET to prevent empty-string overwrites

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-02-01 | Elevation of Privilege | GET /Admin/Settings | mitigate | `[Authorize(Policy="AdminOnly")]` at AdminController class level; `AdminOnly` policy implemented in `AdminHandler.cs`; non-admin receives 302/403 (verified by SETT-07 integration test) |
| T-10-02-02 | Tampering | POST /Admin/Settings (CSRF) | mitigate | `[ValidateAntiForgeryToken]` on Settings POST action; `@Html.AntiForgeryToken()` in form; matches every other POST in AdminController |
| T-10-02-03 | Tampering | OmphalosUrl field injection (user supplies malicious URL) | mitigate | `[Url]` DataAnnotation on SettingsViewModel.OmphalosUrl validates URL format server-side; ModelState.IsValid gate in POST action; `type="url"` provides client-side hint |
| T-10-02-04 | Information Disclosure | OmphalosSharedSecret leakage via page source | mitigate | GET action does NOT populate model.OmphalosSharedSecret; `type="password"` on input field prevents browser rendering of value; `autocomplete="off"` prevents password manager capture |
| T-10-02-05 | Denial of Service | Overly long OmphalosUrl or secret input | mitigate | `[StringLength(2000)]` on OmphalosUrl and `[StringLength(500)]` on OmphalosSharedSecret; validated before SaveSettingsAsync is called |
</threat_model>
