## Layout Contract

### Page Shell

Use the existing `modern-card` pattern exactly as in `Users.cshtml` and `Quests.cshtml`:

```html
<div class="card modern-card">
    <div class="card-header modern-card-header">
        <h2 class="mb-0">
            <i class="fas fa-envelope-open-text text-info me-2"></i>
            Email Stats
        </h2>
    </div>
    <div class="card-body modern-card-body">
        <!-- content -->
    </div>
</div>
```

### Stat Cards Row

Four Bootstrap column cards in a single `<div class="row g-3 mb-4">` row:

- Each card: `<div class="col-md-6 col-xl-3">` (2-wide on tablet, 4-wide on desktop)
- Card inner: `<div class="card modern-card text-center p-3">`
- Icon: `<i class="fas fa-{icon} fa-2x mb-2 text-{color}"></i>`
- Count value: `<div class="fs-1 fw-bold text-{color}">{{count}}</div>`
- Label: `<div class="small text-muted">{{label}}</div>`

| Stat | Icon | Color class | Label text |
|------|------|-------------|-----------|
| Sent | `fa-paper-plane` | `text-primary` | Sent (last 30 days) |
| Delivered | `fa-check-circle` | `text-success` | Delivered |
| Bounced | `fa-exclamation-triangle` | `text-warning` | Bounced |
| Failed | `fa-times-circle` | `text-danger` | Failed |

### Refresh Button

Place below the stat card row, left-aligned or right-aligned — right preferred (`d-flex justify-content-end`):

```html
<div class="d-flex justify-content-end mt-3">
    <a href="/Admin/EmailStats?force=true" class="btn btn-primary">
        <i class="fas fa-sync-alt me-2"></i>Refresh
    </a>
</div>
```

`?force=true` query param signals the controller to clear `IMemoryCache` before fetching — no separate POST endpoint (D-08).

### Last Updated Line

Below stat cards, above Refresh button:

```html
<p class="small text-muted mt-2 mb-0">
    <i class="fas fa-clock me-1"></i>
    Stats as of {{timestamp}} — refreshes every 5 minutes
</p>
```

### Alert Banners

Rendered **instead of** the stat cards row when a degraded state is detected. Use `modern-card .alert` styling already defined in `site.css`:

**Missing API key (D-05):**
```html
<div class="alert alert-warning" role="alert">
    <i class="fas fa-exclamation-triangle me-2"></i>
    <strong>ResendApiKey not configured</strong> — add it to <code>appsettings.json</code> to enable stats.
</div>
```

**API call failure (D-06):**
```html
<div class="alert alert-danger" role="alert">
    <i class="fas fa-times-circle me-2"></i>
    <strong>Could not fetch email stats</strong> — Resend API returned an error. Check your API key and try again.
</div>
```

Both alert banners replace the stat card row. The page header, card shell, and Refresh button still render (allowing the admin to retry).

## Interaction Contract

| Interaction | Behaviour |
|-------------|-----------|
| Page load (normal) | GET `/Admin/EmailStats` — controller checks cache; serves cached counts if fresh, otherwise calls Resend API |
| Page load (forced) | GET `/Admin/EmailStats?force=true` — controller clears the cache key before fetching; returns fresh counts |
| Refresh button click | Navigates to `/Admin/EmailStats?force=true` via anchor (no JavaScript required) |
| Unauthenticated access | Redirected to Login by `"AdminOnly"` policy — no special UI needed |
| Non-admin access | 403 or redirect — handled by existing policy; no special UI needed |

No JavaScript required for this page. All interactions are standard GET navigations.
