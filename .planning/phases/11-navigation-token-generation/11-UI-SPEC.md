## Color

Bootstrap 5 semantic color palette — no custom hex overrides for this phase.

| Role | Bootstrap Class | Hex (Bootstrap 5.3 default) | Usage |
|------|----------------|----------------------------|-------|
| Dominant (60%) | `bg-white` / `bg-body` | `#ffffff` | Page background, card surfaces |
| Secondary (30%) | `bg-light` / card defaults | `#f8f9fa` | Card backgrounds, sidebar cards |
| Accent (10%) | `btn-warning` | `#ffc107` | "Open Session Notes" button only |
| Primary action | `btn-primary` | `#0d6efd` | "Manage Quest" button (existing — do not change) |
| Secondary action | `btn-secondary` | `#6c757d` | "View Public Page" button (existing — do not change) |
| Destructive | `btn-danger` | `#dc3545` | Not used in this phase |

Source: CONTEXT.md "Claude's Discretion" — `btn-warning` or `btn-info` to distinguish from primary buttons. Decision: **`btn-warning`** is chosen because:
1. `#ffc107` is the existing focus ring / accent color in `site.css` (`.form-control:focus` border-color and the shop header gold accent) — it is the app's established secondary accent.
2. `btn-warning` creates clear visual differentiation from the `btn-primary` "Manage Quest" button sitting directly above it in the DM Controls card.
3. `btn-info` (`#0dcaf0`) has no established presence in this codebase and would introduce a new, unexplained color.

Accent reserved for: "Open Session Notes" button (Details page DM Controls card), "Open Session Notes" button (Manage page Session Notes card). These are the only two uses of `btn-warning` in this phase. Do not use `btn-warning` for any other new element.

## Component Inventory

This phase adds four UI elements. All must match existing patterns exactly.

### 1. OmphalosNavItem — Navbar Dropdown Item

**Location:** `_Layout.cshtml` DM dropdown `<ul class="dropdown-menu">`, after the "Edit My Profile" `<li>` (currently line 80), before the closing `</ul>` (currently line 82).

**Rendered HTML (Default.cshtml):**
```html
<li>
    <a class="dropdown-item" href="@Model.OmphalosUrl"
       target="_blank" rel="noopener noreferrer">
        <i class="fas fa-external-link-alt me-2"></i>Open Omphalos
    </a>
</li>
```

**Insertion point in `_Layout.cshtml`:**
```razor
@* After the "Edit My Profile" </li>, before </ul> *@
<li><hr class="dropdown-divider"></li>
@await Component.InvokeAsync("OmphalosNavItem")
```

**Visibility gate:** ViewComponent returns `Content(string.Empty)` when `!settings.IsConfigured`. No HTML is emitted — the divider `<hr>` must be inside the ViewComponent's Default.cshtml, NOT in `_Layout.cshtml` directly, to avoid rendering an orphaned divider when integration is disabled.

**Revised Default.cshtml (with divider inside ViewComponent):**
```html
<li><hr class="dropdown-divider"></li>
<li>
    <a class="dropdown-item" href="@Model.OmphalosUrl"
       target="_blank" rel="noopener noreferrer">
        <i class="fas fa-external-link-alt me-2"></i>Open Omphalos
    </a>
</li>
```

**Layout.cshtml insertion (no divider here):**
```razor
@await Component.InvokeAsync("OmphalosNavItem")
```

Icon: `fas fa-external-link-alt` — signals "opens external link in new tab". This is more semantically accurate than `fa-book-open` for the navbar link (which opens Omphalos root, not session notes).

Pattern match: identical structure to existing `<li><a class="dropdown-item" ...>` items in the same dropdown.

### 2. "Open Session Notes" Button — Details Page (DM Controls card)

**Location:** Inside the existing `modern-card-body` of the "DM Controls" card, after the "Manage Quest" anchor, still inside `<div class="card-body modern-card-body">`.

**Rendered HTML:**
```html
@if ((bool)(ViewBag.ShowOmphalosButton ?? false))
{
    <a href="@Url.Action("LaunchOmphalos", "Quest", new { id = Model.Quest?.Id })"
       class="btn btn-warning w-100 mt-2">
        <i class="fas fa-book-open me-2"></i>
        Open Session Notes
    </a>
}
```

**Button spec:**
- Class: `btn btn-warning w-100 mt-2`
- `mt-2`: 8px top margin to separate from "Manage Quest" button above
- `w-100`: full-width, matching "Manage Quest" button
- Icon: `fas fa-book-open` — conveys "session notes / notes document"
- Label: "Open Session Notes"
- `href`: `@Url.Action("LaunchOmphalos", "Quest", new { id = Model.Quest?.Id })`
- No `target="_blank"` — the LaunchOmphalos action issues a server-side redirect; the browser follows it in the same tab

Pattern match: mirrors `btn-primary w-100` of "Manage Quest" button, same card, same layout.

## Visibility Gating

All Omphalos UI elements share a single gate: `settings.IsConfigured` (i.e., `IsEnabled && !empty(OmphalosUrl) && !empty(OmphalosSharedSecret)`).

| Element | Gate Expression | Evaluated In |
|---------|----------------|-------------|
| "Open Omphalos" navbar link | `settings.IsConfigured` | `OmphalosNavItemViewComponent.InvokeAsync()` |
| "Open Session Notes" (Details) | `ViewBag.ShowOmphalosButton` | `QuestController.Details` GET |
| "Open Session Notes" (Manage) | `ViewBag.ShowOmphalosButton` | `QuestController.Manage` GET |

`ViewBag.ShowOmphalosButton` evaluation: `settings.IsConfigured && currentUser != null && (isQuestDm || isAdmin)`

When any gate condition is false: zero HTML is emitted for that element. No empty containers, no disabled buttons, no placeholder text.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Navbar link label | Open Omphalos |
| Quest button label (both pages) | Open Session Notes |
| New sidebar card title (Manage page) | Session Notes |
| Empty state | Not applicable — elements are simply not rendered when integration is unconfigured |
| Error state | Not applicable — `LaunchOmphalos` returns 404 (handled by existing error page); no inline error message |
| Destructive confirmation | None — no destructive actions in this phase |

Copy rationale:
- "Open Omphalos" (navbar): destination-focused label. DMs know what Omphalos is; naming the destination is clearer than naming the action.
- "Open Session Notes" (quest buttons): action + object label. Describes what happens on the other side of the redirect — the DM will land on the session notes view for this quest.
- "Session Notes" (card title): concise noun that labels the card's purpose.

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| External link indication | `fas fa-external-link-alt` icon on the "Open Omphalos" navbar item signals new-tab behavior visually |
| `rel="noopener noreferrer"` | Required on `target="_blank"` link (NAV-02) — prevents tab-napping and referrer leak |
| Button is an `<a>` tag | Acceptable — Bootstrap `btn` on `<a>` with valid `href` is keyboard-accessible (Enter key activates) |
| No `aria-` additions needed | All elements use standard Bootstrap patterns which carry appropriate ARIA roles implicitly |
