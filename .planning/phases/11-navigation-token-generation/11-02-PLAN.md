**1. Create EuphoriaInn.Service/Components/OmphalosNavItemViewComponent.cs** (new directory + file)

ASP.NET Core MVC auto-discovers ViewComponents by convention — class name ends `ViewComponent`, placed anywhere in the assembly. No `Program.cs` registration needed.

```csharp
using EuphoriaInn.Domain.Interfaces;
using Microsoft.AspNetCore.Mvc;

namespace EuphoriaInn.Service.Components;

public class OmphalosNavItemViewComponent(IAdminSettingService adminSettingService) : ViewComponent
{
    public async Task<IViewComponentResult> InvokeAsync()
    {
        var settings = await adminSettingService.GetSettingsAsync();
        if (!settings.IsConfigured)
            return Content(string.Empty);   // renders nothing — no HTML emitted when unconfigured
        return View(settings);              // passes IntegrationSettings to Default.cshtml
    }
}
```

`IAdminSettingService` is in `EuphoriaInn.Domain.Interfaces` — same namespace used throughout the Service layer. `ViewComponent` base class is in `Microsoft.AspNetCore.Mvc`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| _Layout.cshtml → OmphalosNavItemViewComponent | ViewComponent DB call on every layout render (Scoped service — one hit per request) |
| Default.cshtml → OmphalosUrl | OmphalosUrl is admin-configured; rendered directly as href — potential open redirect if admin enters a malicious URL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-08 | Tampering | Default.cshtml href="@Model.OmphalosUrl" | accept | OmphalosUrl is set by Admin role only (AdminOnly policy on Settings page — Phase 10). Admin-level trust is appropriate; no additional sanitization needed for private group app |
| T-11-09 | Information Disclosure | Default.cshtml renders OmphalosUrl in HTML | accept | OmphalosUrl is not sensitive — it is the public Omphalos base URL. OmphalosSharedSecret is [JsonIgnore]-protected and never passed to views |
| T-11-10 | Denial of Service | OmphalosNavItemViewComponent DB call per layout render | accept | IAdminSettingService is Scoped (single DB call per HTTP request); EF Core result is not cached but overhead is negligible for a group app with low concurrency |
| T-11-11 | Spoofing | ViewBag.ShowOmphalosButton null cast in views | mitigate | `(bool)(ViewBag.ShowOmphalosButton ?? false)` defensive pattern prevents NullReferenceException if action fails to set the flag; button simply does not render |
</threat_model>
